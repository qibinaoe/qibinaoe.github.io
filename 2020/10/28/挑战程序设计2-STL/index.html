以上代表注意点

  1. STL中队列和栈，pop不返回值，而是要先front获取元素后再pop出来

  2. STL中pair是用于保存数值的结构体模板，注意是结构体不是类，因此访问的时候，对于第1个元素通过.first访问，第2个元素通过.second访问。同时make_pair()用于生成一对数值，注意是圆括号不是尖括号

vector 向量

vector属于动态数组或者说边长数组，即添加元素的时候能够增加长度；而必须事先指定长度，只能容纳一定数量元素的叫做静态数组

注意

begin()返回的是指向元素的迭代器，而end()返回的是指向末尾元素的后一个位置的迭代器，不指向元素

知识点总结

  • vector在内存中是分配一段连续的内存空间进行储存，(分配多大？),因此才有.capacity()这个函数来访问总的容量，当超过该空间后需要重新分配一段新的空间。
  • 动态的进行push_back()和pop_back()操作
  • 像数组一样，通过[]操作符进行访问元素
    但是
  • 在内部插入效率低，需要挪动空间
  • 只能在尾部进行插入和删除，不能在头部进行push 和 pop

总之

vector适合高效率随机访问的场景，不适合频繁的在内部插入删除操作

list 双向链表

感想

感觉list是vector的超集，很多vector有的功能，list都有，除此之外，相比于vector,list的`insert(p,x)`和`erase(p)`的时间复杂度是O（1）,而vector是 O(n)

问题

  1. 为什么list的在指定位置插入和删除比vector快
    • 答：因为vector插入删除需要挪动内存空间的元素，而list只需要修改相邻结点的指向
  2. list能够通过[]直接进行访问吗？
    • 答：不能

这里书本有误？

注意

  • 以int举例，begin()返回迭代器的是list<int>::iterator
  • 获得迭代器元素用解引用符*，比如*it

知识点总结

  • list底层是带头结点的双向循环链表实现（参考CSDN博客STL中vector和list的区别）,因此字在内存中不是连续空间
  • list不支持随机访问
  • list插入和删除效率高，只需要改变相邻的两个结点的指向
  • 可以在链表的两端进行push和pop操作

总之

list适合大量插入删除操作的场景，不适合随机访问。

搜索

迭代器 iterator

迭代器是一种对象，用于对STL容器进行迭代操作。对于任何种类的容器，迭代器顺序访问元素使用的语法相同。在处理数组元素时，可以当作C/C++指针来用

迭代器基本运算符

  • ++ 让迭代器指向下一个元素
  • ==,!= 判断两个迭代器是否指向同一个位置并返回结果
  • = 将右侧的值带入左侧迭代器所引用元素的位置
  • * 返回该位置的元素

迭代器函数

  • begin() 返回指向容器开头的迭代器
  • end() 返回指向容器末尾（最后一个元素的下一个位置）的迭代器

二分搜索

STL在二分搜索提供了binary_search、lower_bound、upper_bound函数。这几个函数的使用的前提是数组是有序且非递减。包含在头文件#include<algorithm>中

lower_bound

lower_bound(起始地址，结束地址，查询值),注意是左开右闭。返回一个迭代器(指针)，该指针指向第一个大于或等于（或者说不小于）指定值的元素的地址。
例如lower_bound(A,A+14,3)，指定数组A的头指针以及距离头指针14的位置，一般为数组的长度A+size。如果A为vector，则可以使用A.begin(),A.end()来指定范围。

upper_bound

upper_bound(起始地址，结束地址，查询值),注意是左开右闭。返回一个迭代器(指针)，该指针指向第一个大于指定值的元素的地址。

binary_search

binary_search(起始地址，结束地址，查询值),注意是左开右闭。找到元素返回真，找不到返回假.

distance()用于返回两个指针间的距离
