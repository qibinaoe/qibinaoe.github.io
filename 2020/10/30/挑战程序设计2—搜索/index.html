线性搜索

线性搜索是从开头顺序遍历各元素来判断是否与目标值相等，效率低但是适用于任何形式数据。

线性搜索改进

在线性搜索的过程中引入一个特殊“标记”，在处理大规模数据时会有常数倍的性能提升。例如本题中，将搜索目标key放在数组末尾作为标记，当while遍历的末尾时，就必定会返回相等，但此时意味着没找到。对于性能的提升在于，相比for循环需要两次判断。1次在判断是否到底末尾，1次在判断是否匹配目标。而加入“标记”，只需要判断是否匹配到目标。

二分搜索

二分搜索适用于已经提前按照关键字进行升序排列的数组，由于每一次搜索，数组范围都会减半，因此其时间复杂度为O(logn)

注意点

  • 二分搜索中需要规定搜索范围的左边和右边，左边left为搜索范围的第一个元素，而右边right为最后一个元素的后一个位置。
  • 因此初始化的时候left = 0，right = n，对于判断的位置middle为middle = (left + right)/2。
  • 在一次判断后，如果我们判断的关键字小于当前位置A[middle] > key，说明key小，要在数组前部分搜索，此时需要改变right位置来缩小范围，而right为搜索范围的最后一个元素的后一个位置，而此时middle已经不在我们的搜索范围了。因此left = middle
  • 在一次判断后，如果我们判断的关键字大于当前位置A[middle] < key，说明key大，要在数组后部分搜索，此时需要改变left位置来缩小范围，而left为搜索范围的第一个元素，而此时middle已经不在我们的搜索范围了。因此right = middle+1,而不是right = middle，虽然这样也能正常运行，但是不是正版的二分搜索
  • 判断条件left < right。
    • 假设我们要搜索的关键词小于数组所有元素，则left一直为0，而right在不断的缩小，在最后一次判断中，right注定是1,而middle为(0+1) /2= 0,因此我们判断了A[0]的位置，再进行一次则right == left跳出循环.
    • 假设我们要搜索的关键词大于数组所有元素，则right一直为n，而left在不断的变大，在最后一次判断中，left注定是n-1,而middle为(n-1+n) /2= n-0.5 = n-1,因此我们判断了A[n-1]的位置，再进行一次则right == left == n跳出循环.
  • 对于middle = (right + left) / 2这句语句执行的位置应该为while循环的第一句

散列法

散列法是一种搜索算法，能够根据个元素的值来确定储存位置，从而实现数据的告诉搜索。而其中的散列表是一种数据结构，能够对关键字的数据集合高效地执行动态插入、搜索和删除操作。

与链表区别

  • 链表在删除和搜索操作中，时间复杂度为O(n)
  • 散列表在不考虑发生冲突的情况下，时间复杂度为O(1)

散列表

  • 散列函数 h(k)： 根据k的值求得数组T的下标的函数
  • 散列值： 散列函数的返回值
  • 数组T的长度为m，则散列值的范围为0 ~ m-1。因此函数内需要使用到求余运算
  • 冲突：不同的key对应到同一个散列值
开放地址法

开放地址法是解决冲突的手段之一，这里介绍的是双散列结构里的开放地址法。

在双散列结构中一旦发生冲突，则会调用第二个散列函数求散列值:H(k) = h(k,i) = (h1(k)+i*h2(k)) mod m

i是发送冲突后计算下一个散列值的次数。当h1(k)计算第一个下标冲突后，则会下标移动h2(k)个位置，知道不发送冲突为止。

注意

  • 下标每次移动的h2(k)个位置应该与数组长度m互质，否则会出现无法生成下标的情况。因此可以让m为质数，而h2(k)为小于m的数字

  • 在搜索的时候，判断散列表中不存在要搜索的元素时，是通过T[j] == NIL || i >= m的条件
