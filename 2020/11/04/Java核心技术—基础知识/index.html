零碎小知识

  • Java应用程序中的全部内容都必须放在类中

  • 源代码的文件名必须与公共类的名字相同（区分大小写）

  • 运行已编译的程序时，JVM从指定类中的main方法的代码开始执行

  • 根据Java语言规范，main方法必须声明为public、静态的

  • Java中所有函数都是某个类的方法

  • Java中的main方法必须有一个外壳(shell)类

  • System.out.println("We"),System.out对象调用println方法

  • .用于调用方法object.method(parameters)

  • 注释3种形式

    • // hello
    • /* */
    • 自动生成文档,以/**开始，*/结束

  • Java是强类型语言，共有8种基本类型

    • 整型
      • int 4字节 -2 147 483 648 ~ 2 147 483 647(刚过20亿)
      • short 2字节 -32768 ~ 32767
      • long 8字节
      • byte 1字节 -128~127
      • 长整型数值有一个后缀L或l(如4000000000L)
      • 十六进制数值有一个前缀0x或0X(如0xCAFE)
      • 八进制有一个前缀0
      • Java7开始，前缀0b或0B表示二进制数，如(0b1001)是9
      • Java7开始，可以为数字字面量加下划线(1_000_000)
      • 注：Java没有任何无符号(unsigned)形式的int、long、short或byte
      • Byte.toUnsignedInt(b)
    • 浮点类型
      • float 4字节 有效位数为6~7位
      • double 8字节 有效位数位15位
      • float类型有一个后缀F或f
      • 没有后缀的浮点类型总是默认位double
      • double后缀D或d
      • 十六进制表示浮点数值0.125 = 2-3 = 0x1.0p-3。尾数采用十六进制、指数采用十进制、指数的基数是2
    • 字符类型char(用于表示Unicode编码的代码单元)
      • 2个字节
      • 十六进制表示\u0000~\uFFFF
      • Unicode转义序列在解析代码之前得到处理
      • 小心注释中的\u会报错
        • // \u000A is a new line
        • // look inside c:\users
      • char类型描述了UTF-16的一个代码单元
      • 总之：char是用UTF-16编码表示的Unicode码点的代码单元
    • 表示真值的boolean类型

      • true false

      • 整型值和布尔值之间不能进行相互转换

        • if (x = 0)在Java中不能编译通过，因为整数表达式不能转换成布尔值j
      •   while(pos--){ //ERROR 不能这样
      after = after.next;
  }
    • 注：const是Java保留的关键字，但是目前并没有使用，必须用final定义常量

  • 枚举类型

    • enum Size {SMALL, MEDIUM, LARGE, EXTRA_LARGE};
    • Size s = Size.MEDIUM;
    • Size s = null;

  • 枚举类型的变量只能储存这个类型声明中给的枚举值或者特殊值null,null表示这个变量没有设置任何值

  • 算术运算符

    • / 参与/的运算的两个操作数都是整数时，表示整数除法： 15/2 = 7
    • / 参与/的运算的操作数存在浮点数时，表示浮点除法： 15.0/2 = 7.5
    • 整数被0除，会抛出异常java.lang.ArithmeticException: / by zero
    • 浮点数被0除，得到的结果是无穷大或NaN结果Infinity(如果是负数，则结果是-Infinity)

  • strictfp标记的方法，则方法内的所有指令都使用严格的浮点计算

  • Java没有幂运算，因此需要借助Math类的pow方法:Math.pow(x,a)表示xa

  • floorMod 是Java的取模运算。 %是Java的取余运算，详细见问题2

  • StrictMath得到一个完全可预测的结果比运行速度更重要的时候

  • Math提供一些方法使得整数运算有更好的安全性

    • multiplyExact
    • addExact
    • …

  • 位运算符

    • &&和||用于布尔值会采用短路，但是&和|用于布尔值不会采用短路
    • >> << 用于位模式左移或右移。 >>>运算符会用0填充高位，注意没有<<<

  • 字符串

    • 码点和代码单元

      • char数据类型给是一个采用UTF-16编码表示的Unicode码点的代码单元。

        • 码点是一个编码表中的某个字符对应的代码值
        • 每个字符(char)用16位表示代码单元
          • 注意：在C和C++中，windows的32位机和64位机下，char占1个字节，这跟Java不一样。Java的char类型占2个字节(16bit)

      • 因此每个码点可能由多个代码单元组成

      • length()方法返回的是采用UTF-16编码表示的给定字符串所需要的代码单元

      • codePointCount()返回实际长度，即码点数量

      • charAt(n)返回位置n的代码单元

    • String类实现了CharSequence的接口

      • StringBuilder 字符串构造器
      • StringBuffer
        • StringBuffer效率比StringBuilder稍低
        • StringBuffer允许采用多线程的方式添加或删除字符
        • 如果是在单个线程执行的话，应该使用StringBuilder

问题回答

  1. 取余和取模区别，

  • 取余rem 取模mod

  • 被除数 ➗ 除数 ＝ 商

  • 取余运算在计算中，使得商值向0的方向取整(换句话是说向0的方向舍弃小数位)

  • 取模运算在计算中，使得商值向负无穷的方向取整(换句话是说向负无穷的方向舍弃小数位)

  • 结论

    • 当被除数和除数符号相同时(同正或同负)，取余和取模结果相同
    • 当被除数和除数符号相反时(一正一负)，取余和取模结果不同

  • 前提： 对于一个除法(a = bq + r),如果 a不能被b整除（a÷b)，那么能够得到两种解，一种是bq > a，使得r < 0。另一种是bq< a，使得r > 0。

  • 例子

    • mod(7,3) = 1, rem(7,3) = 1
    • mod(-7,-3) = -1, rem(7,3) = 1
    • mod(7,-3) = -2, rem(7,-3) = 1
      • 两种解: (-2,1) , (-3,-2)
    • mod(-7,3) = 2, rem(-7,3) = -1
      • 两种解: (-2,-1) , (-3,2)

  • Java中%是取余。Math.floorMod是取模
