Java核心技术—对象与类

发表于 2020-11-11 更新于 2022-06-08 分类于 Java 阅读次数： 本文字数： 7.4k 阅读时长 ≈ 12 分钟

Java核心技术——第四章 对象与类

  • 面向对象设计(Object-oriented programming ,OOP)

  • 对象的三个主要特性

    • 对象的行为——可以对对象完成哪些操作，或者可以对对象应用哪些方法？
    • 对象的状态——当调用那些方法时，对象会如何相应？
      • 状态:每个对象都保存着描述当前状况的信息
      • 状态可以随时间而发生改变，但是改变不是自发的
      • 对象状态的改变必须通过调用方法实现（否则破坏了封装性）
    • 对象的标识——如何区分具有相同行为和状态的不同对象？

  • 面向对象的三个基本特征

    • 封装
    • 继承
    • 多态

  • 创建类的实例(instance): 由类class(构造对象的模板或蓝图)构造construct对象的过程

  • 封装(encapsulation)的关键在于程序只能通过对象的方法与对象数据进行交互，换句话说，绝对不能让类中的方法直接访问其他类的示例字段(instance field)

  • 对别对象和对象变量的概念

    • Date deadline; 这是一个对象遍历啊你给，不是一个对象，并且没有引用任何对象
    • 对象变量初始化的方法
      1. 引用一个新构造的对象 deadline = new Date();
      2. 引用一个已存在的对象 deadline = birthday;
    • 对象变量没有实际包含对象，而是引用一个对象
    • new操作符返回值是一个引用
    • 所有的Java对象都存储在堆中。一个对象包含另一个对象变量时，它只是包含着另一个堆对象的指针
    • Java中必须使用clone方法来获得对象的完整副本

  • UTC(Coordinated Universal Time) 国际协调时间

  • GMT(Greenwich Mean Time) 格林尼治时间

  • 访问器方法(accessor method) 只访问对象而不修改对象的方法

  • 修改器方法(mutator method) 访问对象后，对象的状态发生改变

  • 一个源文件(.java)只能由一个公共类，但可以任意数目的非公共类

    • 在编译的时候会分别为每个类生成.class文件

  • 构造器

    • 构造器结合new运算符调用
    • 不能对一个已经存在的对象调用构造器来达到重新设置实例字段的目的
    • 构造器没有返回值

  • 不要在构造器中定义与实例字段同名的局部变量

  • 隐式参数与显示参数

    • employee.raiseSalary(1000);
      • raiseSalary方法有两个参数
        1. 隐式参数： employee出现在方法前的Employee类型对象（方法调用的目标或接收者）
        2. 显示参数： 1000位于方法方法名后面括号中的数值
    • 在每个方法中，关键字this指示隐式参数
      • this.salary += raise;

  • Java中所有的方法都必须在类的内部定义(不同于C++),但这不表示它们是内联方法。是否将某个方法设置为内联方法是Java虚拟机的任务。即时编辑器会监视那些简短的、经常调用而且没有被覆盖的方法调用，并进行优化。

  • 访问器方法

    • 不要编写返回可变对象引用的访问器方法
    • 如果需要返回一个可变对象的引用，应该先对其克隆(clone)再返回。return (Date)hireDay.clone();

  • 基于类的访问权限

    • 方法可以访问调用这个方法的对象的私有数据
    • 一个方法可以访问所属类的所有对象的私有数据(可读可修改)

  • final实例字段

    • 实例字段定义为final
      • 这些字段必须在构造对象时初始化，即每一个构造器执行之后，这些字段的值已经设置，并且以后不能再修改这个字段
    • final修饰符对于类型为基本类型或不可变类(类中所有方法都不会改变其对象)的字段(String)
    • final修饰可变的类会造成混乱(StringBuilder)，因为final只表示储存在final修饰的变量的对象引用不会再知识另一个不同的对象，但是这个对象可以更改

  • 静态字段

    • 一个字段定义为static,则每个类只有一个这样的字段
      • 这个类的所有实例都共享这个static修饰的静态字段
      • 即使没有这个对象，静态字段仍然存在
      • 静态字段属于类，不属于任何单个对象
    • 对于非静态的实例字段，每个对象都有自己的副本

  • 静态常量

    • public static final double PI = 3.14159265358979323846;
    • public static final PrintStream out = ...;
    • 在System类中，有个setOut方法能够修改System.out为不同的流，这个方法能够修改final修饰变量的值的原因是因为这是个原生方法,不是在Java语言实现的，绕过Java语言的访问控制机制。

  • 静态方法

    • 不在对象上执行的方法 Math.pow(x, a), 由于不使用任何对象，因此它没有隐式参数
    • 静态方法没有this参数
    • 静态方法不能访问实例字段，但是可以访问静态字段
    • 非静态方法也也可以访问静态字段 这里的static可以省略，但这样一来就得通过对象引用来调用方法

  • 工厂方法

    • 静态工厂方法构造对象
      • 无法命名构造器
        • 希望有两个不同的名字得到不同的实例
      • 使用构造器，无法改变所构造对象的类型

  • 每一个类都可以有一个main方法

    • 对类进行的单元测试的技巧

  • 方法参数

    • 按值调用(call by value)
      • 接受调用者提供的值
      • 方法不能修改按值传递的变量的值
    • 按引用调用(call by reference)
      • 接受的是调用者提供的变量地址
      • 方法可以修改按引用传递的变量的值
    • Java总是采用按值调用

      • 方法得到的是所有参数值的一个副本，方法不能修改传递给它的任何参数变量的内容

      • 方法参数类型

        • 基本数据类型(数字、布尔值)
        • 对象引用(注：对象引用也是按值传递，不是按引用传递)

      • 但是通过对象引用能够修改对象的参数状态，因为方法得到的是对象引用的一个副本，原来对象引用和副本都引用同一个对象

      • 证明对象引用不是按引用传递

  • 对象构造

    • 重载(overloading)
      • 相同名字，不同参数。(不仅适用于构造器，Java允许重载任何方法)(注意区分重载(overload)和覆盖(override)区别)
    • 重载解析(overloading solution)
      • 查找匹配的过程
    • 方法的签名: 方法名+参数类型
      • 注意，返回值不是签名的一部分

  • 默认字段初始化

    • 构造器没有显示为字段设置初值，会被自动赋为默认值
      • 数值为0
      • 布尔值为false
      • 对象引用为null
    • 注意：方法中的局部变量必须明确地初始化，否则会报错

  • 无参数构造器

    • 写类的时候没有编写构造器，会提供一个无参数构造器
      • 所有实例字段设置为默认值
    • 仅当类没有任何其他构造器的时候，才会得到一个默认的无参数构造器
      • 如果类提供了至少一个构造器，但是没有提供无参数的构造器，那么构造对象时如果不提供参数就是不合法

  • 显式字段初始化

    1. 类定义中直接为任何字段赋值

      • 执行构造器之前先完成赋值操作

      • 希望将某个字段设置为同一值

    2. 利用方法调用初始化一个字段

  • this关键字调用另一个构造器

    • 构造器的第一个语句形如this(...)，构造器将调用同一个类的另一个构造器

  • 初始化块(initialization block)

    • 首先运行初始化块，再运行构造器的主体部分
    • 通常将初始化代码放在构造器里
    • 建议将初始化块放在字段定义之后

  • 调用构造器的具体处理步骤

    1. 如果构造器第一行调用了另一个构造器，则基于所提供的参数执行第二个构造器
    2. 否则
      1. 所有数据字段初始化为其默认值
      2. 按照类声明中出现的顺序，执行所有字段的初始化方法和初始化块
    3. 执行构造器主题代码

  • 静态字段初始化

    1. 提供初始值
      • private static int nextId = 1;
    2. 静态的初始化块、

  • 包

    • 一个类可以使用所属包的所有类，以及其他包的公共类
    • 在包中定位类是编译器(compiler)的工作。
    • 类文件中的字节码总是使用完整的包名引用其他类
    • 在Java中，通过显式地给出类名(java.util.Date)，可以完全避免使用import机制

  • 静态导入

    • import语句允许导入静态方法和静态字段，而不只是类
    • 源文件顶部添加import static java.lang.System.*;
      • 就可以使用System类的静态方法和静态字段，不必加类名前缀
      • out.println("Godd bye");
      • exit(0);

  • 包访问

    • public: 可以由任意类使用
    • private: 只能由定义它们的类使用。（提示，变量必须显示的标记为private，否则将默认为包访问，会破坏封装性）
    • 没有指定public和private的(类、方法、变量): 可以被同一个包的所有方法访问

  • 编译器查找文件

    • 如果引用了一个类而没有指定这个类的包(不是这样Date deadline = new java.util.Date();),它首先查找包含这个类的包(通过import指令，来确定其中是否包含这个类)
    • 如果找到了一个以上的类，就会产生编译时错误(因为完全限定类名必须是唯一的，与import次序无关)

  • java.lang包总会默认导入

  • JAR(Java Archive)

    • JAR文件使用ZIP格式组织文件和子目录，可以使用任何ZIP工具查看JAR文件
    • Java归档文件
    • 可以包含类也可以包含图片和声音等其他类型的文件
    • 创建JAR文件
      • jar cvf jarFileName file1 file2...
        • 例如jar cvf CalculatorClasses.jar *.class icon.gif
    • jar程序的选项类似Unix tar命令的选项
    • 清单文件(mainfest)
      • 包含在JAR文件当中，一个特殊的META-INF的子目录的
      • 清单文件被命名为MAINFEST.MF
      • 清单文件被分成多个节
        • 第一节称为主节(main section)
        • 随后的条目用来指定命名实体的属性

  • 文档注释 javadoc

    • javadoc抽取信息，为一下特性编写注释

      • 模块
      • 包
      • 公共类与接口
      • 公共的和受保护的字段
      • 公共的和受保护的构造器及方法

    • 注释放置在所描述特性的前面

    • 注释以/**开始，以*/结束

    • 自由格式文本(free-form text)

    • 标记以@开始，如@since、@param

    • 自由格式文本第一句应该是一个概要性的句子，javadoc工具将这些句子抽取出生成概要页

    • 自由格式文本中可以使用html修饰符

      • <em> ... </em> 强调
      • <strong> ... </strong> 着重强调
      • <img ...> 图片
      • {@code ...}等宽代码

    • 类注释

      • 类注释必须放在import语句之后，类定义之前

    • 方法注释

      • 方法必须放在所描述的方法之前
      • 除了通用标记，还能使用以下标记
        • @param variable description
          • 给当前方法的参数部分添加条目
          • 可以占据多行
          • 一个方法的所有@param必须放在一起
        • @return description
          • 给当前方法添加返回部分
          • 可以跨多行
        • @throws class description
          • 表示方法可能抛出异常

    • 字段注释

      • 只需要对公共字段(通常指静态常量public static final)建立文档

    • 通用注释

      • @since text
        • 建立一个"since"(始于)条目
        • "text"可以是引入这个特性的版本任何描述
          • @since 1.7.1
      • @author name
        • 作者条目
        • 可以使用多个@author标记
      • version text
        • 版本条目
        • 对当前版本的任何描述
      • @see和@link
        • 可以使用超链接，链接到javadoc文档的相关部分或外部文档
        • @see reference
          • 将在"see also"部分增加一个超链接
          • 可用于类和方法
          • reference可以有以下选择
            1. package.class#feature label
              • 例如@see com.horstmann.corejava.Employee#raiseSalary(double)
              • 包名或包名和类名都可以省略，这样会定位于当前包或当前类
            2. <a href="...">label</a>
              • @see <a href="www.horstmann.com/corejava.html">The Core Java home page</a>
            3. "text"
              • @see "Core Java 2 volume 2"
        • {@link package.class#feature lable}

    • 包注释

      • 需要在每个包目录中添加一个单独文件
        1. 提供package-info.java的Java文件
        • 该文件包含一个初是的以/**和*/界定的javadoc注释
        • 后面是一个package语句
        1. 提供package.html的HTML文件
          • 会抽取<body></body>之间所有文本

  • 类设计技巧

    • 一定要保证数据私有
      • 绝对不要破坏封装性
    • 一定要对数据进行初始化
      • 不要依赖系统的默认值
      • 自己提供默认值
      • 所有构造器中设置默认值
    • 不要再类中使用过多的基本类型
      • 用其他类替换使用多个相关的基本类型
    • 不是所有的字段都需要单独的字段访问器和字段更改器
    • 分解有过多指责的类
    • 类名和方法名要能够体现出它的职责
    • 优先使用不可变类
      • 并发更改：多个线程试图同时更新一个对象
      • 如果类是不变的，可以安全地在多个线程间共享其对象
