Java核心技术——第五章 继承

类、超类和子类

  • 继承(inheritance)

    • "is-a"关系是继承的一个明显特征

  • Java用关键字extends表示继承

  • Java中所有继承都是公共继承，没有C++中的私有继承和保护继承

    • 新类派生于一个已存在的类

      • 已存在的类称为
        • 超类 superclass
        • 基类 base class
        • 父类 parent class
      • 新类称为
        • 子累 subclass
        • 派生类 derived class
        • 孩子类 child class

    • 通过扩展超类定义子类的时候，只需要支出子类与超类的不同之处。

  • 覆盖(override)方法

    • 超类中的有些方法对于子类不适用需要提供新的方法覆盖

      • 注意：子类不能直接访问超类的私有字段(private)，因此可以通过访问器来获取，但是如果我们重写的方法就是访问器的话，需要加上super，否则会一直循环调用自身以致程序崩溃

      • super和this概念不同

        • this是对象的引用(隐式参数)
        • super不是对象的引用
          • 不能将值super赋给另一个对象变量
          • super只是一个指示编译器调用超类方法的特殊关键字

  • 子类构造器

    • 由于子类的构造器不能访问父类的私有字段，所以必须通过一个构造器来初始化私有字段
    • 利用super语法来调用父类的构造器
      • 注意：使用super调用构造器的语句必须是子类构造器的第一条语句
    • 如果子类构造器没有显示地调用超类的构造器，将自动调用超类的无参数构造器
    • 如果超类没有无参数构造器，并且子类的构造器没有显示地调用超类的其他构造器，Java编译器会报错
    • this与super
      • this两个含义
        1. 指示隐式参数的引用
        2. 调用该类的其他构造器
      • super
        1. 调用超类的方法
        2. 调用超类的其他构造器
    • 调用构造器的语句只能作为另外一个构造器的第一条语句出现
    • 多态与动态绑定
      • 一个对象变量(例如下面代码中的e)可以指示多种实际类型的现象叫做多态(polymorphism)
      • 在运行时能够自动地选择适当的方法，称为动态绑定(dynamic binding)
      • 虽然下面的for each循环中的e声明为Employee,但是e既可以引用Employee类型的对象，也可以引用Manager类型的对象
      • 虚拟机知道e实际引用的对象类型，因此能够正确地调用相应的方法
      • Java中，动态绑定时默认的行为
      • 如果不希望让一个方法是虚拟的，可以将它标记为final

    • 继承层次(inheritance hierarchy)：由一个公共超类派生出来的所有类的集合

    • Java不支持多重继承

    • 多态

      • "is-a"规则可以用来判断是否应该将数据设计为继承关系，其指出子类的每个对象也是超类的对象(经理都是员工)，但是反之则不然

      • "is-a"规则另一种表述：替换原则(substitution principle)

        • 出现超类对象的任何地方都能用子类对象替换

      • Java对象变量是多态(polymorphic)的

  • 子类覆盖超类方法

    • 子类方法不能低于超类方法的可见性
      • 如果超类方式是public，那么子类的方法必须也要声明为public。如果子类方法遗漏掉了public修饰符，则编译器会报错

  • 方法调用过程

    • 调用x.f(args),其中x为C类的一个对象
      1. 编译器查看对象的声明类型和方法名。编译器会一一列举C类中所有方法名为f的方法和其超类中所有名为f且可访问的方法
      2. 编译器确定方法调用中提供的参数类型。在所有名为f的方法中，如果存在一个与所提供参数类型完全匹配的方法，就调用该方法。这个过程称为重载解析(overloading resolution)。如果编译器没有找到与参数类型匹配的方法，或者发现经过类型转换后有多个方法与之匹配，编译器会报错。
      3. 静态绑定(static binding):如果是private方法,static方法,final方法或构造器，则编译器可以准确知道应该调用哪一个方法；动态绑定:要调用的方法依赖于隐式参数的实际类型，必须要运行时使用动态绑定
      4. 假设程序运行并采用动态绑定调用方法，则虚拟机调用与x所引用对象的实际类型对应的那个方法。如果该类没有定义这个方法，则在父类中找。虚拟机预先为每个类计算了一个方法表(method table)，其中列出了所有方法的签名和要调用的实际方法。在真正调用方法时，虚拟机仅查找该表、

  • final类

    • 需求：如果想阻止人们用某个类来定义子类

    • 不允许扩展的类称为final类(定义类的时候使用final修饰符)

    • 例如阻止人们派生Executive类的子类

  • final方法

    • 类中的某个特定的方法也可以声明为final
    • 子类不能覆盖这个方法
    • final类中的所有方法自动成为final方法

  • 字段也能声明为final，final字段表示构造对象后就不能再改变值了。final类只有其中的方法自动成为final，不包括字段

  • 将方法或类声明为final的主要原因是确保它们在子类不会改变语义

    • Calendar类中的getTime()和setTime()方法声明为final。意味着Calendar类设计者负责实现Date类与日历状态的转换，不允许子类添乱
    • String类为final类，如果有一个String引用，则它引用一定是String对象

  • 内联(inlining)

  • 如果一个方法没有被覆盖(重写)而且很短，则编译器能够对它进行优化处理

  • 类的对象引用的强制类型转换

    • 用一对圆括号将目标类括起来，并放置在需要转换的对象引用之前

    • 使用对象引用的强制类型转换的唯一原因： 要在，暂时忽视对象的实际类型之后，使用对象的全部功能

      • 暂时忽视的情况：在staff中，某些元素是普通员工(Employee)，因此经理元素(Manager)需要暂时忽视它的实际类型
      • 要使用对象全部功能： 将staff数组中引用Manager对象的元素复原成Manager对象

    • 每个对象变量都有一个类型

      • 类型描述这个变量所引用的以及能够引用的对象类型

    • 一个值存入变量时，编译器会检查是否承诺过多

      • 子类的引用赋值给超类的变量，是允许的

    • 超类的引用赋值给子类变量时，是不可以的，承诺过多，必须进行强制类型转换

    • 如果试图在继承链上，进行向下的强制类型转换，并且谎报对象内容，会产生ClassCastException的异常

    • 在强制类型转换之前，先查看是否能够转换成功，使用instanceof操作符来判断

    • String c = (String) staff[1];会产生编译错误

    • 总之

      • 只能在继承层次内进行强制类型转换
      • 在超类强制转换成子类之前，应该使用instanceof进行检查

    • 因为实现多态性的动态绑定机制，能够自动找到正确的方法，很多情况不需要进行强制类型转换

    • 只有在使用子类(如Manager)中特有的方法才需要进行强制类型转换

  • 抽象类

    • 包含一个或多个抽象方法的类，类本身必须声明为抽象的

    • 抽象类还可以包含字段和具体方法

    • 通过引用一个公共超类，将方法放在继承层次结构中更高层

    • 而当高层的类中的方法需要根据实际情况实现时(例如这里的getDescription())，可以定义成抽象方法，就不需要实现这个方法，留到继承它的类去实现。

    • 扩展抽象类的两种情况

      • 子类保留抽象类的部分方法，或者是所有抽象方法都仍未定义，因此子类也应该标记为抽象类
      • 子类实现定义的全部方法，那么子类就不是抽象的了

    • 即使不含有抽象方法，也可以将类声明称抽象类

    • 抽象类不能实例化

    • 可以创建具体子类的对象(父类是抽象的，但是本身不是抽象的)、

    • 可以定义抽象类的对象变量

      • Person p = new Student("dog","math")
      • 这个定义为抽象类的对象变量只能引用非抽象子类的对象

    • 使用抽象类一个应用就是能够通过直接访问子类中实现抽象类的方法，方便处理

      • 如果这里将Person超类中的getDescription省略掉的话，就不能在变量p上调用getDescription方法了。

  • 受保护访问 protected

    • 希望限制超类中的某个方法只允许子类访问
    • 希望允许子类的方法能够访问超类的某个字段
    • Java中，保护字段只能由同一个包中的类访问
      • 避免滥用保护机制，不能通过派生子类来访问受保护字段

  • 访问控制修饰符

    • 仅对本类可见——private
    • 对外部完全可见——public
    • 对本包和所有子类可见——protected
    • 对本包可见——默认，不需要修饰符

  • Object:所有类的超类

    • Java中每个类都扩展了Object

    • 可以使用Object类型的变量引用任何类型的对象

    • Java中只有基本类型(primitive type)不是对象

      • 数值、字符、布尔类型都不是对象

  • 所有的数组类型，不管是对象数组还是基本类型的数组都扩展了Object类

    • 编写equals方法的建议

      • 显示参数命名为otherObject，稍后需要将它强制转换程另一个名为other的变量

    • hashCode方法

      • 散列码(hash code)是由对象导出的一个整型值(可正可负)
      • 如果x和y是两个不同的对象，x.hashCode()与y.hashCode()基本上不会相同

    • String类通过下列方法计算散列码 s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]

      • 每个对象(没有重写的情况下)都有一个默认的散列码(Object类实现的)，其值由对象的储存地址得出

      • 如果重新定义了equals方法，就必须为用用户可能插入散列表的对象重新定义hashCode方法

        • 要合理地组合示例字段的散列码，以便能够让不同对象产生的散列码分布均匀

    • null安全的方法Objects.hashCode

  • 自己写一个hashCode()

  • Objects.hash()方法能够对传入各个参数调用Objects.hashCode,并组合
    • 其底层用的是Arrays.hashCode()实现
    • 注意：equals与hashCode的定义必须相容，如果x.equals(y)返回true,那么x.hashCode()就必须与y.hashCode()返回相同的值

  • 泛型数组列表

    • Java允许在运行时确定数组的大小

    • ArrayList 数组列表

      • 类似数组，但是能够在添加或删除元素时，自动地调整数组容量

      • ArrayList是一个有类型参数(type parameter)的泛型类(generic class)

        • 用一对尖括号将类名括起来追加到ArrayList后面来指定数组列表保存的元素对象的类型

            • 赋值给一个变量
            • 传递给某个方法
            • 从某个方法返回

  • 对象包装器和自动装箱

    • 所有的基本类型都有一个与之对应的类(包装器 wrapper)

      • Integer
      • Long
      • Float

      • Byte

    • 提供参数数量可变的方法(变参(varargs)方法)

    • System.out.printf("%d",n);

    • System.out.printf("%d %s",n, "widgets");

      这里的 ...是Java代码的一部分，表明这个方法可以接受任意数量的对象,除format外

      Object ... args其实是一个Object[] 数组，保存所有其他参数，如果调用者提供的是基本类型的值，则会自动装箱

  • 枚举类

    • public enum Size {SAMLL, MEDIUM, LARGE, EXTRA_LARGE}
      • 声明定义的类型是一个类，它有四个实例，只有这四个实例，所以不可能再构造新的对象
    • 比较两个枚举类型的值，直接使用==就可以
    • 可以为枚举类型添加构造器，方法，字段
    • 所有枚举类型都是Enum类的子类，继承了很多方法

      • toString

        • Size.SMALL.toString() 返回"SMALL"

      • toString的逆方法是静态方法valueOf

        • Size s = Enum.valueOf(Size.class,"SMALL");

      • values()方法返回一个包含全部枚举值的数组

        • System.out.println(Arrays.toString(Size.values())); //[SMALL, MEDIEUM, LARGE, EXTRA_LARGE]

      • ordinal()方法返回声明中枚举常量的位置，位置从0开始计数

      • int compareTo(E other)

        • 返回结果是 self.ordinal - other.ordinal

  • 反射

    • 反射库(reflection library)提供丰富且精巧的工具集，可以用来编写能够动态操纵Java代码的程序
    • 反射(reflective) 能够分析类能力的程序
    • 反射机制
      • 在运行时候分析类的能力
      • 在运行时检查对象，例如编写一个适用于所有类的toString方法
      • 实现泛型数组操作代码
      • 利用Method对象，这个对象类似C++的函数指针
    • Class类

      • Java运行时，系统始终为所有对象维护一个运行时类型标识。这个信息会跟踪每个对象所属的类

        • 保存这个信息的是Class类

      • Object类中getClass()方法返回一个Class类型的实例

        • Class对象描述一个特定类的属性
          • getName()方法返回类的名字，会带上包名
          • System.out.println(new Random().getClass().getName()); // java.util.Random

  • 静态方法forName()获得类名对应的Class对象

    `forName`方法会抛出一个*检查型异常(checked exception)*,因此需要提供一个*异常处理器(exception handler)*
    • 在启动时，包含main方法的类被加载，它会加载所有需要的类，这些被加载的类又要加载它们需要的类。对于大型应用程序将会花费很长时间
    • 如果T是任意的Java类型(或void关键字)，则T.class代表匹配的类对象

  • Class对象实际上表示的是一个类型，可能是类，也可能不是类

    • 虚拟机为每个类型管理一个唯一的Class对象，因此可以用==运算符实现两个类对象的比较

      • if(e.getClass() == Employee.class)
      • 与e instanceof Employee不同，如果是某个子类的话，测试失败

    • Class类型的对象，可以用它来构造类的实例

      • 用getConstructor方法得到Constructor类型的对象

    • 使用newInstance方法来构造一个实例

      • 异常
        • 非检查型(unchecked)异常,例如越界错误、访问null引用
        • 检查型(checked)异常

  • 继承的设计技巧

    • 将公共操作和字段放在超类
    • 不要使用受保护的字段
    • 使用继承实现"is-a"关系
      • 如果不满足这种关系，就不要使用，例如父类有的某些字段和方法，在子类中不允许存在
    • 除非所有继承的方法都有意义，否则不要使用继承
    • 在覆盖方法时，不要改变预期的行为
    • 使用多态，而不要使用类型信息
    • 不要滥用反射
