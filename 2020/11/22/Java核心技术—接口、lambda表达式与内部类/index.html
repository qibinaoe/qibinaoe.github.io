Java核心技术—接口、lambda表达式与内部类

发表于 2020-11-22 更新于 2022-06-08 分类于 Java 阅读次数： 本文字数： 16k 阅读时长 ≈ 27 分钟

Java核心技术 第六章 接口、lambda表达式与内部类

接口

接口不是类，而是对希望符合这个接口的类的一组需求

接口用来描述类应该做什么，而不指定它们具体应该如何做

举例，Arrays类中的sort方法实现了对对象数组进行排序，但是当然不是任意的对象数组都能够进行排序，因此，如果想使用sort方法进行排序，则对象所属的类必须实现Comparable接口

实现Comparable接口的类，都需要包含compareTo方法，这里Comparable接口为一个泛型类型，因此实现该接口的时候需要带上类型

Comparable也可以使用不带类型参数的，但是这样在实现的方法里就有Object类型的参数，需要手动的进行强制转换

  • 接口中所有方法都是自动是public方法，可以不提供关键字public,但是提供也是可以的

  • 在实现接口的类中，重写的方法必须明确声明为public,否则编译器会认为是包可见性而报错。

  • 接口可以定义常量

  • 接口没有实例字段

实现接口的步骤

  1. 将类声明为实现给定的接口
  2. 对接口中的所有方法提供定义

书上问题：

为什么不能直接在类中直接提供一个compareTo方法，而必须实现Comparable接口呢？

回答

Java是一种强类型(strongly typed)语言，调用方法的时候，编译器要检查该方法确实存在。如果一个类实现了Comparable的接口，可以确保拥有compareTo方法，

书上问题：

对于继承了实现Comparable接口的子类之间比较的问题

解决办法：

  • 如果不同子类中的比较有不同的含义(意思就是同一个类中可以进行比较，但是与另一个类无法进行比较)，应该将属于不同类的对象之间的比较视为非法的。每个compareTo方法开始时进行检测 if(getClass() != other.getClass()) throw new ClassCastException();
  • 如果存在一个能够比较子类对象的通用算法，可以在超类中提供一个compareTo方法，并将这个方法声明为final

接口不能用new运算符实例化一个接口

能够声明接口的变量，同时该接口变量必须引用实现了这个接口的类对象

使用instanceof来检查一个对象是否实现了某个特定的接口

接口链： 从通用性较高的接口扩展到专用性较高的接口

  • 接口通过extends来实现接口的扩展

  • 接口中的方法都自动设置为public

  • 接口中的字段总是public static final

    • 注：在接口中可以显示的标记出来，也可以不标记出来

  • 每个类只能有一个超类

  • 每个类能够实现多个接口

    • 实现多个接口用逗号隔开: class Employee implements Cloneable, Comparable

书上问题：

为什么不直接继承抽象类？

回答：

Java每个类只能扩展一个类

书上问题：

为什么Java不采用多重继承(multiple inheritance)

回答：

多重继承会让语言变得复杂和效率低下

允许在接口中添加静态(statics)方法，但是有违将接口作为抽象规范的初衷

通常做法：将静态方法放在伴随类中，例如表中库的中接口和实用工具类成对出现，Collection/Collections 和Path/Paths

默认方法

用default修饰符来标记默认方法

可以为接口实现默认方法

如果一个接口(interface)中有default标识的方法，实现这个接口的类可以不用实现default标识的方法，但是还是一定要实现不是默认方法，也不是静态方法。

默认方法的好处就是保证了源代码兼容(source compatible)，接口演化(interface evolution)

默认方法里面可以调用其他方法

默认方法冲突

情景：一个接口中有默认方法，同时在超类或者另一个接口也有定义同样的方法

解决规则

  • 超类优先

    • 如果超类提供具体的方法，同名且有相同参数类型(签名相同)的默认方法会被忽略

  • 接口冲突的情况

    • 如果一个接口提供默认方法，另一个接口提供了同名且参数类型相同的方法(无论是否是默认参数)，必须覆盖这个方法解决冲突

      • 重新写这个方法

      • 使用其中一个接口实现的方法，这里要加super才能够使用这个方法

如果一个类implements一个接口，但是不实现里面的方法，就需要将该类定义为抽象类(abstract)

不要让静默方法重新定义Object类中的方法，因为所有类都继承Object类，根据类优先的原则，静默方法里面定义的Object类的方法是没有效果的。

接口与回调

回调(callback)： 指定某个特定事件发生时应该采取的动作

对于事件发送时应该采取的动作，Java不是通过传入一个函数，而是通过传入一个对象。调用该对象实现的方法即为动作。那么怎么知道应该调用哪个动作呢？因此该类应该去实现某个接口(interface)，通过重写该接口中的方法来明确这个动作。

例如一个定时器Timer类，通过一定时间间隔调用一个方法

这是Timer的构造器，可以看到其中一个参数是ActionListener listener

对于ActionListener,这是一个接口，因此如果想要实现这个功能，需要写一个类去继承该接口，并实现这个方法，传入Timer，这样就能够正确的调用所需要触发的方法。

Comparator 接口

需求：如果我们想按照自己的需求对于字符串(String)进行比较排序，例如按照长度进行排序，并不关心字典顺序。那么让String类实现Comparable接口是行不通的。

原因有两个

  1. 最根本的原因是String是final类，不能够继承来重写方法
  2. 即使我们可以重写String实现Comparable接口的compareTo方法，但是对于新的需求的时候，还得去改代码，这样不太方便和现实，因为compareTo方法只能写一个

因此Arrays.sort()有第二个版本

第一个参数是要被排序的数组，第二个参数即为一个比较器(comparator)。该比较器是实现了Comparator接口的类的实例

因此我们可以这样实现一个比较器的类

在传入Arrays.sort()时，比较器的参数需要实例化，因为Arrays.sort()内部是通过调用对象的方法进行比较的:if (c.compare(a[runHi++], a[lo]) < 0)

对象克隆

  • 用=来为一个包含对象引用的变量建立副本时，只是将原变量和副本指向同一个对象的引用

  • Object类中有一个声明为protected的clone方法

    • 浅拷贝
      • Object类中这个clone方法默认是浅拷贝，即没有克隆对象中引用的其他对象
      • 如果对象中的所有数字段都是数值或其他基本类型，拷贝字段没有问题
      • 如果对象包含子对象的引用，拷贝字段就会得到相同子对象的另一个引用
        • 原对象和克隆的对象会共享信息
      • 如果原对象和浅克隆对象共享的子对象是不可变的类(例如 String)，则共享安全
      • 如果共享子对象一直包含不变的常量、没有更改其方法改变它、没有方法生成引用，则也是安全
    • 深拷贝(deep copy)
      • 克隆对象的时候同时克隆所有子对象

对于克隆对象需要确定几点

  1. 默认的clone方法是否满足要求
  2. 是否可以在可变的子对象调用clone来修补默认的clone方法
  3. 是否不该使用clone

如果需要使用clone方法有以下步骤

  1. 实现Cloneable接口
  2. 重新定义clone方法，并指定public访问修饰符

注意

  • 如果要使用clone的方法，虽然重写了clone方法，却没有实现Cloneable的接口，会抛出异常java.lang.CloneNotSupportedException

  • 将修饰符改成public的原因是为了在所有的方法都能够克隆对象

  • 标记接口(tagging interface) 记号接口(marker interface)

    • 标记接口不包含任何方法
    • 唯一作用是允许在类型查询中使用instanceof
      • 例如if(obj instanceof Cloneable)...

Cloneable里面没有方法，clone方法是从Object类继承的

实现浅拷贝

  1. 实现Cloneable接口
  2. clone定义为public
  3. 调用super.clone()

Object类中返回类型是Object类，根据协变返回类型，可以在子类中指定正确的返回类型

实现深拷贝

实现深拷贝除了浅拷贝的基础上实现的基本类型，还需要克隆对象中可变的实例字段

  • 所有数组类型都有一个公共(public)的clone方法，可以建立一个新数组，包含原数组所有元素的副本

lambda表达式

lambda表达式是一个可传递的代码块，可以在以后执行一次或多次

场景：一个代码块传递到某个对象中，这个代码块会在将来某个时间调用

lambda表达式形式

  • 参数
  • 箭头(->)
  • 一个表达式
    • 如果代码要完成的计算无法放在一个标傲世中，可以像写方法一样放在{}中，并包含显示的return语句

如果lambda表达式没有参数，仍要提供空括号

如果可以推导出一个lambda表达式的参数类型，则可以忽略其类型

如果方法只有一个参数且这个参数类型可以推导出，则可以省略小括号

  • 函数式接口(functional interface)

    • 对于只有一个抽象方法的接口，需要这种接口对象时，可以提供一个lambda表达式

  • Java中，lambda表达式所能做的也只是转换为函数式接口

方法引用(method reference)

表达式System.out::println是一个方法引用，它指示编译器生成一个函数式接口的实例，覆盖这个接口的抽象方法来调用给定的方法

使用方法引用可改写成

用::运算符分隔方法名与对象或类名

  1. object::instanceMethod
  2. Class::instanceMethod
  3. Class::staticMethod

第一种情况下，方法引用等价于向方法传递参数的lambda表达式:System.out::println — x->System.out.println(x)

第二种情况下，第一个参数变成方法的隐式参数：Arrays.sort(planets,String::compareToIgnoreCase); — (x,y)->x.compareToIgnoreCase(y)

第三种情况下，所有参数都传递到静态方法:Math::pow — (x,y)->Math.pow(x,y)

注：只有当lambda表达式的体只调用一个方法而不做其他操作的时候，才能改写成方法引用

方法引用和lambda表达式区别之一在于：lambda只有在调用的时候，才会抛出异常。方法引用在构造的时候就会抛出异常

方法引用中可以使用this参数，例如this::equals 等同于x -> this.equals(x)

方法引用使用super参数， super::instanceMethod

构造器引用

构造器引用的方法名为new，例如Person::new是Person构造器的一个引用

如果有多个Person构造器，编译器会从上下文推导出符合类型的构造器

数组类型可以建立构造器引用 int[]::new

数组构造器能够克服Java无法构造泛型类型T的数组 Person[] people = stream.toArray(Person[]::new)

变量作用域

  • lambda表达式就是闭包

lambda表达式有3部分

  1. 一个代码块
  2. 参数
  3. 自由变量的值，这里指非参数而且不在代码中定义的变量

lambda表达式可以捕获外围作用域中的变量值，要求所捕获的值是明确定义的同时值不会改变的变量

lambda表达式所捕获的变量必须是事实最终变量(effectively final)，即该变量初始化之后就i不会再为它赋新值。

lambda表达式的体与嵌套块具有相同的作用域：适用命名冲突和遮蔽

lambda表达式中使用this关键字，是指创建这个lambda表达式的方法的this参数

自己设计接口，其中只有一个抽象方法，用@FunctionalInterface注解来标记这个接口。有两个好处

  1. 如果无意增加另一个抽象方法，编译器会产生错误信息
  2. javadoc也指出该接口时一个函数式接口

内部类

内部类(inner class)是定义在另一个类中的类

使用内部类的原因

  1. 内部类可以对同一包中的其他类隐藏
  2. 内部类方法可以访问定义这个类的作用域中的数据，包括原本的私有数据
  • C++的嵌套类与Java的内部类
    • Java的内部类的对象会有一个隐式引用，指向实例化这个对象的外部类对象，通过这个指针可以访问外部对象的全部状态
    • Java静态内部类没有这个指针，相当于C++的嵌套类

内部类位于外部类的内部，不代表每个外部类都有一个内部类的实例字段。

内部类的方法可以访问自己的数据字段，也可以访问创建它的外围类对象的数据字段(上面的meet)

  • 内部类的对象总有一个隐式引用，指向创建它的外部类的对象。
    • 这个引用在内部类的定义是不可见的(即没有显示的定义这个隐式引用)
    • 外围类的引用是在内部类的构造器中设置
      • 编译器会修改所有内部类的构造器，添加一个对应外围类引用的参数

内部类可以是私有的(private)

常规类可以有包可见性或(没有访问修饰符)公共可见性(public)

内部类中，表示外围类的引用:OuterClass.this

可以采用以下语法更加明确的编写内部类对象的构造器 outerObject.new InnerClass(construction parameters)

当内部类是公共访问的时候，可以这样构造。

在外围类的作用域之外，这样引用内部类 OuterClass.InnerClass

内部类声明的所有静态字段必须是final,并初始化为一个编译时常量

内部类不能有static方法

内部类是一个编译器现象，与虚拟机无关。编译器会把内部类转换成常规的类文件，用$分割外部类名与内部类名，虚拟机对此一无所知

例如 OuterClass$InnerClass

与直接写一个常规类来访问某一个类，内部类拥有更大的访问权限。内部类可以访问外部类的私有数据，而常规类不能访问。

局部内部类

当内部类在外部类中名字只出现一次的时候(只是在这个方法中创建这个类型的对象时使用了一次)，可以将内部类改写成局部内部类

即在一个方法中局部的定义这个类

声明局部类时，不能有访问说明符(public或private),局部类的作用域被限定在声明这个局部类的块中。局部类对外部世界完全隐藏，外部类的其他代码也不能访它。

局部类不仅能够访问外部类的字段，还可以访问局部变量，但要求局部变量必须是事实最终变量(effectively final)，即一旦赋值就不会改变。

编译器检测局部内部类(LocalInnerClass)中对局部变量(truelove)的访问，为每一个变量建立相应的实例字段，并将局部变量复制到构造器，从而能够初始化这些实例字段。

匿名内部类 anonymous inner class

局部内部类更进一步就是匿名内部类。如果只想创建这个类的一个对象，甚至不需要为类指定名字(名字是类似class Name implements interface中的Name,不是指变量名)。

这个语法的意思是创建一个类的新对象，这个类实现了ActionListener接口，需要实现的方法actionPerformed在括号{}内定义

语法格式

SuperType可以是接口，如ActionListener,内部类要实现该接口，同时如果是接口，就不能有任何构造参数

SuperType可以是类，内部类要扩展这个类

由于构造器的名字必须与类名相同，匿名内部类米欸有类名，所以匿名内部类不能有构造器。而这里的construction parameters构造参数要传递给超类superclass构造器

构造一个类的新对象与构造一个扩展了那个类的匿名内部类的对象之间的区别在于小括号后面是否有大括号

可以为匿名内部类提供对象初始化块

静态内部类

静态内部类：为了把一个类隐藏在另一个类的内部，并不需要内部类有外围对象的一个引用。将内部类声明为static就不会生成那个引用

只有内部类可以声明为static

当静态方法中构造一个内部类，则该内部类必须是静态的

  • 与常规内部类不同，静态内部类可以有静态字段和方法
  • 接口中声明的内部类自动是static和public
