Java核心设计 第八章 泛型程序设计

泛型程序设计

泛型设计(generic programming) : 编写的代码可以对多种不同类型的对象重用

类型参数(type parameter) ArrayList<String> files = new ArrayList<>();

在Java增加泛型类之前，泛型程序设计是用继承实现的，存在两个问题

  • 当获取一个值时，必须进行强制类型转换
  • 没有错误检查，可以向数组列表中添加任何类的值

在使用泛型类和类型参数后

  • 编译器可以通过类型参数来知道返回值的类型，因此不需要进行强制类型转换
  • 在添加元素（ArrayList中的add()）时，编译器知道插入元素的类型，比Object类型安全。编译器可以检查，防止插入错误类型的对象

通配符类型(wildcard type)

泛型类

泛型类(generic class) :有一个或多个类型变量的类

  • 类型变量 T ，用尖括号(<>)括起来，放在类名后面

  • 泛型类可以有多个类型变量 public class Pair<T,U> {...}

  • 类型变量在整个类定义中，用于

    • 指定方法的返回类型
    • 字段的类型
    • 局部变量的类型

类型变量使用大写字母

  • 变量E表示集合的元素类型
  • K和V分别表示表的键和值的类型
  • T（和相邻的字母U和S）表示”任意类型“

泛型类相当于普通类的工厂

  • 用具体的类型替换类型变量来**实例化(instantiate)**泛型类型

泛型方法

泛型方法可以在普通类中定义也可以在泛型类中定义

**类型变量**的位置

  • 放在修饰符（这里指public static）的后面
  • 放在返回类型的前面（这里的返回类型是T）

调用一个泛型方法

  • 可以把具体类型包为在尖括号中，放在方法名前面
    • String middle = ArrayAlg.<String>getMiddle("Jogn","CAt","Dog");
  • 大多数情况下，编译器有足够信息推断出想法，将参数的类型和泛型类型T进行匹配进行推断，可以省略<String>类型参数
    • String middle = ArrayAlg.getMiddle("Jogn","CAt","Dog");

类型变量的限定

有时，类或方法需要对类型变量加以约束(例如保证方法中的某些方法能够正常调用)，可以通过对类型变量T设置一个**限定(bound)**来实现

使用关键字extends

记法 <T extends BoundingType>

  • T表示限定类型（BoundingType）的子类型（subtype）
  • T和限定类型可以是类也可以是接口
  • 一个类型变量(T)或通配符可以有多个限定(Bound)
    • T extends Comparable & Serializable
    • 限定类型（BoundingType）用”&”分隔
    • 类型变量（T)用逗号分隔
  • 在限定类型中可以拥有多个接口作为超类型，但最多有一个限定可以是类。
    • 如果一个类作为限定，它必须是限定列表中的第一个限定
