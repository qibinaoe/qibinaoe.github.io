配置密钥登陆

通过密钥登陆就省得每次登陆都输入密码了

原理介绍：

公钥添加到服务器的某个账户上

私钥放在客户端

客户端利用私钥即可完成认证并登陆

  • 没有私钥，任何人都无法通过 SSH 暴力破解密码来远程登录到系统
  • 如果将公钥复制到其他账户甚至主机，利用私钥也可以登录

参考来自菜鸟教程-设置 SSH 通过密钥登录

操作大体流程：

  • 主机A（客户端）创建公钥私钥
  • 将公钥复制到主机B（被登陆机）的指定用户下
  • 主机A使用保存私钥的用户登录到主机B对应保存公钥的用户

参考来自Linux密钥登录原理和ssh使用密钥实现免密码登陆

用WinSCP与我们服务器相连

Linux scp 命令用于 Linux 之间复制文件和目录。
scp 是 secure copy 的缩写, scp 是 linux 系统下基于 ssh 登陆进行安全的远程文件拷贝命令。
scp 是加密的，rcp 是不加密的，scp 是 rcp 的加强版。

git搭建

参考资料

  • 开发人员学Linux(9)：CentOS7编译安装Git-2.13.1及使用
  • 带你跳过各种坑，一次性把 Hexo 博客部署到自己的服务器

首先查看是否已经安装git

git --version

在root权限下安装git

方式一：通过yum安装，但是版本有点低（1.8.3.1）

（-y代表默认yes，就不用在安装的时候键入y）

sudo su

git -y install git

因此卸载了，通过源码编译去安装新版本的git

卸载git：

yum remove git

方式二：源码编译安装

安装git第三方库的依赖

yum install asciidoc docbook2X xmlto texinfo sgml2xml autoconf openjade -y

yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker -y

记得这里要卸载git的，因为上面的安装又把低版本的git下载下来了

yum remove git

将源码压缩包下载至/root目录下

这里是git各版本的目录git下载目录

cd /root

wget https://mirrors.edge.kernel.org/pub/software/scm/git/git-2.29.2.tar.gz

下载后解压

tar -zxvf git-2.29.2.tar.gz

进入目录

cd /root/git-2.29.2

编译安装

make configure

./configure --prefix=/usr/local/git

make all doc

make install install-doc install-html

配置git环境变量

修改/etc/profile文件

export GIT_HOME=/usr/local/git
export PATH=$PATH:$GIT_HOME/bin

使配置文件生效

source /etc/profile

添加git运行时的user和usergroup

groupadd git
useradd git -g git

设置git用户密码

passwd git

将git用户暂时添加进sudoers中，方便接下来命令执行

修改/etc/sudoers文件

vim /etc/sudoers

将其添加进去

git ALL=(ALL) ALL

强制保存:wq!

在git用户下，通过ssh登录本机

su git

ssh git@localhost

输入yes并输入密码

通过exit退出

以git身份执行ssh-keygen,并一直回车

ssh-keygen

通过如下命令创建authorized_keys文件

touch ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys

将生成的公钥文件id_rsa.pub的内容添加到authorized_keys文件中去（authorized_keys用于存放本机免密码登录的用户公钥）：

cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

在git用户下创建仓库

cd /home/git

mkdir -p project/blog

mkdir repo

cd repo

git init --bare blog.git

cd blog.git/hooks

创建钩子

vim post-receive

输入

#!/bin/sh git --work-tree=/home/git/project/blog --git-dir=/home/git/repo/blog.git checkout -f

保存后修改权限

chmod +x post-receive exit

切换到root用户

sudo su

chown -R git:git /home/git/repo/blog.git

接下来在本地自己的电脑将ssh的密钥上传到服务器

ssh-copy-id -i ~/.ssh/id_rsa.pub git@123.123.123.123

然后在本地自己的电脑测试git

git clone git@server_ip:/home/git/repo/blog.git

在本地如果克隆git clone时，出现错误

bash: git-upload-pack: command not found

将git安装目录的git-upload-pack链接到/usr/bin/git-upload-pack

ln -s /usr/local/git/bin/git-upload-pack /usr/bin/git-upload-pack

ln -s /usr/local/git/bin/git-receive-pack /usr/bin/git-receive-pack

禁止git用户shell登录权限，只允许git clone和git push

查看git-shell路径

which git-shell

将上一步显示出来的路径添加到/etc/shells中，得先切换到root用户

修改/etc/passwd文件

vim /etc/passwd

拉到最下面，将原本的

git:x:1001:1001::/home/git:/bin/bash

改成

git:x:1001:1001:,,,:/home/git:/bin/git-shell

最后记得将git移除sudoers文件中
