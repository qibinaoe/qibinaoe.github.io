maven作用

开发和管理

  • 管理jar文件
  • 自动下载jar文件及其相关文档、源代码
  • 管理jar直接的依赖
    • 即该jar中所会用到的其他jar
  • 编译程序：将java编译成class
  • 测试代码
  • 打包文件：打包成jar或war
  • 部署项目

项目的构建

构建是面向过程的，即完成项目代码的编译、测试、运行、打包、部署等步骤

maven支持的构建包括

  1. 清理
    • 将之前项目编译的东西删掉，为新编译的代码做准备
  2. 编译
    • 把程序源代码编译成执行代码：java -> class文件
    • 批量：maven可同时编译多个文件，而javac一次编译一个文件
  3. 测试
    • maven可执行测试程序代码
    • 批量：maven可同时执行多个测试代码，同时测试多个功能
  4. 报告
    • 生成测试结果的文件
  5. 打包
    • 将项目中所有的class文件，配置文件等所有资源放在一个压缩文件中
    • java程序生成jar扩展名
    • web应用生成war扩展名
  6. 安装
    • 把打包生成的文件（jar、war）安装到本机仓库
  7. 部署
    • 把程序安装好可以执行
    • 一般不用maven进行部署，会更麻烦

maven核心概念

  • POM(Project Object Model)项目对象模型
    • 文件名为pom.xml
    • 包含项目的基本信息，用于描述项目如何构建，声明项目依赖（如jar）等
  • 约定的目录结构
    • maven项目的目录和文件的位置都是规定的
  • 坐标
    • 唯一的字符串，用来表示资源
  • 依赖管理
    • 管理项目可以使用的jar文件
  • 仓库管理
    • 资源存放的位置
  • 生命周期
    • maven工具构建项目的过程
  • 插件和目标
    • 执行maven构建的时候用的工具
  • 继承
  • 聚合

仓库

仓库用于存放maven使用的各种插件jar包与我们项目中使用到的jar（第三方工具）

仓库分类

  • 本地仓库：个人计算机文件夹中
  • 远程仓库：互联网上的
    • 中央仓库：最权威的：https://repo.maven.apache.org
    • 中央仓库的镜像：中央仓库的备份
    • 私服：公司内部，在局域网中使用，不对外开发

  2. 私服
  3. 镜像
  4. 中央仓库

pom文件

pom（project object model）:项目对象模型，是一个pom.xml文件

基本信息 解释
modelVersion maven模型的版本，对于maven2和maven3来说，只能是4.0.0
groupId 组织id，一般为公司域名的倒写，例如 1.域名倒写：com.baidu 2.域名倒写+项目名：com.baidu.appolo
artifactId 项目名称、模块名称，对应groupId的子项目
version 项目的版本号。如果项目还在开发，是不稳定版本，通常在版本后带-SNAPSHOT。version使用三位数字标识，例如1.1.0。例子0.0.1-SNAPSHOT
packaging 项目打包的类型，可以是jar,war,rar,ear,pom。默认是jar
依赖 解释
dependencies和dependency maven通过dependency进行jar包的管理。项目构建运行中，使用到的其他jar包称为依赖。使用标签进行配置，而依赖的配置通过坐标来定位
配置属性 解释
properties properties是用来定义一些配置属性的
构建 解释
build 表示与构建相关的配置，例如编译插件的jdk版本
继承 解释
parent 在maven中，如果多个模块都需要声明相同的配置，例如groupId,version,有相同的依赖，或相同的组件配置等，可用parent声明要继承的父工程的pom配置
聚合 解释
modules 在maven的多模块开发中，为统一构建整个项目的所有模块，可以提供一个额外的模块，该模块的打包方式为pom，并且在其中使用modules聚合的其他模块，这样通过本模块就可以一键自动识别模块间的依赖关系来构建所有模块，叫maven的聚合

maven的常用命令

maven的命令就对应于maven的生命周期：编译，测试，报告，打包，安装，部署

maven的命令执行依赖于插件，即一些jar文件

Junit单元测试

junit测试类中的方法，每一个方法都是独立测试的

操作步骤

  1. 在pom.xml加入单元测试依赖
  1. 在maven项目中的src/test/java目录下，创建测试程序
  • 测试类命名为：Test+要被测试的类名
  • 测试方法命名为：test+方法名称

例如测试HelloMaven，则创建测试类TestHelloMaven

测试方法定义规则

  • 方法是public
  • 方法没有返回值
  • 方法名称是自定义的，但推荐为test+方法名称
  • 在方法上面加入@Test

命令说明

maven命令的执行必须在pom.xml所在的目录

  • mvn clean 清理
    • 会删除原来编译和测试的目录，即target目录，但是已经install到仓库的包不会删除
  • mvn compile 编译main/java下的java程序
    • 会在当前目录下生成一个target，里边存放编译主程序之后生成的字节码文件
  • mvn test-compile 编译test/java下的java程序
    • 会在当前目录下生成一个target，里边存放编译测试程序之后生成的字节码文件
  • mvn test 测试
    • 会生成一个目录surefire-reports，保存测试结果
  • mvn package 打包
    • 会编译、编译测试、测试，并且按照pom.xml配置把主程序打包生成jar包或者war包
  • mvn install 安装
    • 会把本工程打包，并且按照本工程的坐标保存到本地仓库中
  • mvn deploy 部署
    • 会把本工程打包，按照本工程的坐标保存到本地库中，并且还会保存到私服仓库中，还会自动把项目部署到web容器中

默认为remote，即从 http://repo1.maven.org/maven2/archetype-catalog.xml路径下载archetype-catalog.xml文件。
