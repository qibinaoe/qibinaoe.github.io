排序算法分类

可以根据元素间是否进行比较进行划分

比较类排序

  • 通过两元素间进行比较，来决定元素间的相对次序，其时间复杂度不能突破O(nlogn)（可根据比较树进行推导），因此也称为非线性时间比较类排序

非比较类排序

  • 不通过两元素间的比较来决定相对次序，因此可以突破比较类排序的时间下界，以线性时间运行，因此也称为线性时间非比较类排序

可以根据比较的数据记录所在位置进行划分

内部排序

  • 数据记录在内存中进行排序

外部排序

  • 因数据记录很大，不能一次性都放入内存，所以在排序的过程中需要访问外存

冒泡排序 Bubble Sort

冒泡排序：重复走访要排序的数列，一次比较两个元素，如果这两个元素顺序错误，则将其交换位置。走访数列的工作进行到没有元素再需要交换为止。

算法步骤

假设排序目标：从左到右，从小到大排列。假设一共n个元素

  1. 从头开始，比较相邻的两个元素，如果第一个元素大于第二个元素（即顺序错误），则交换这两个元素。
  2. 对每一对相邻的元素做相同的操作（比较大小，纠正顺序）。例如第1个元素与第2个元素、第2个元素和第3个元素…第n-1个元素和第n个元素。在这一趟遍历后，会使得最大的元素归位（即到最后一位）
  3. 重复以上操作，直到数组没有再需要交换位置的相邻元素

最快情况

当数据数列已经是按期望有序

最慢情况

当数据数列是反序时

其他话

冒泡排序每一趟遍历，都会有一个未排序的最大值归位

冒泡排序有两层for循环，外层for循环代表一共要遍历多少趟。假设有n个元素，每一趟完成一个未排序的最大值归位，那么在第n-1趟时，前n-1大的数字已经归位了，那最小的那一个元素也肯定就留在原地不需要动了。一次外层循环最多遍历n-1次

内层循环代表相邻元素交换位置的过程。因为每一层遍历，都会有一个最大值归位，所以内层循环不需要每次都遍历到数组末尾，只需要将未排序的最大元素放到相应位置即可。

优化：如果一趟遍历中，没有任何元素进行交换，说明该数组已经有序，可以直接跳出循环，没必要遍历n-1次

选择排序 Selection Sort

选择排序的工作原理为：第一步在未排序的数列中，寻找最小的元素，放置排序序列起始位置，构成排序部分。接下来在后面的未排序部分中，不断选择最小的元素，放置排序部分的末尾。直到所有元素都排序完毕

算法步骤

  1. 初始状态：整个数组都为未排序部分[1,n]
  2. 在第i趟排序中(i=1,2,…n)，前i-1个元素为有序部分[1,i-1]，而第i个位置为此趟排序，要将其转换成有序部分。因此从无序部分[i,n]中选择最小的元素，将其与第i个元素进行交换，至此，前i个元素[1,i]就变成有序部分
  3. 重复n-1趟

插入排序 Insertion Sort

插入排序即为玩扑克牌的时候，整牌的过程。其工作原理为构建有序序列，按顺序对于未排序的数据，在已排序的序列中，从后向前扫描，找到相应的位置并插入

算法步骤

  1. 数据序列的第一个元素认为是有序序列，因此对从第2个元素至最后一个元素（未排序序列）进行插入排序
  2. 取出下一个元素K（第一个未排序元素），在已排序序列中从后向前扫描
  3. 如果扫描到的已排序序列的元素J大于K，则将J元素向后移动一位
  4. 直到找到已排序序列中的元素J小于或等于K的位置
  5. 将K插入到J元素后面

对于小数据，使用插入排序效率高

希尔排序 Shell Sort

希尔排序，亦称缩小增量排序（递减增量算法），是插入排序的更高效的改进版本。

希尔排序是基于插入排序的以下两个性质进行改进

  1. 插入排序对于几乎已经排好的数据操作时，效率高，可以达到线性排序的效率（O(n)）
  2. 但是插入排序一般来说是低效的，因为插入排序每次只能将数据移动一位
    • 希尔排序会优先比较距离较远的元素

希尔排序的基本思想是：先将整个待排序的记录序列分成若干个子序列分别进行直接插入排序，待整个序列中的记录基本有序时，再对整体记录进行一次插入排序

算法步骤

  1. 选择一个增量序列t1, t2, …, tk, 其中 ti > tj (i < j), tk = 1
    • 增量即子序列中两个元素相差的距离
  2. 按增量序列个数k，对序列进行k趟排序
  3. 每趟排序，根据对应的增量ti，将待排序列分割成若干长度为m的子序列，分别对各子表进行直接插入排序。当增量因子为1时（即希尔排序的最后一次排序），将整个序列作为一个表来处理，表长度即为整个序列的长度

间隔序列

希尔排序的间隔序列gap（增量序列，说法不一，意思到位就可），当gn+1 = 3gn+1时，算法复杂度基本维持在O(N1.25)。

如果遇到2的幂指数（1,2,4,8,…）等g=1之前不需要排序的数列，希尔排序的效率会大幅度降低

增量序列的选取，应该尽可能减少不同增量值之间的公共因子，而一种彻底的方法就是保证它们之间两两互素

Pratt序列

pratt = {1，2，3，4，6，8，9，12，16，…，2p3q，…}

希尔排序最多运行O(log2n)

Sedgewick序列

sedgewick = {1，5，19，41，109，209，505，929，2161，3905，8929，…}

通项公式为：9·4k - 9·2k + 1 或 4k - 3·2k + 1

最坏时间复杂度为O(n4/3)，平均时间复杂度为O(n7/6)

归并排序 Merge Sort

归并排序是建立在归并操作的排序算法。归并排序采用分治法（Divide and Conquer），分而治之。其思路为将已有序的子序列合并，得到完全有序的序列。

归并排序实现的两种方法

  • 自上而下的递归
  • 自下而上的迭代

算法步骤

自顶向下归并排序

  1. 申请空间，该空间大小为两个已排序的子序列的大小之和。该空间用于存放合并后的序列
  2. 设定两个指针，初始位置分别为两个已排序的子序列的起始位置
  3. 比较两个指针所指向的元素的大小，选择相对小的元素放入合并的空间，并将指针移动到下一位
  4. 重复步骤3直到某一指针达到序列尾部
  5. 将另一个序列剩下的元素直接复制到合并序列的尾部

自底向上归并排序

  1. 申请空间，该空间大小为未排序数组的长度，用于存放数组序列（作用同自顶向下归并排序）
  2. 定义每次归并元素的长度size
  3. 首先两两归并（每个元素为长度为1的数组，size=1），其次四四归并（将两个大小2的数组归并成一个大小为4的有序数组，size=2），接着八八归并，一直下去
  4. 直到最后一次归并，第二个子数组的长度可能比第一个子数组要小

其他话

归并排序的性能不受输入数据（的无序程度）影响，但是表现比选择排序好，时间复杂度始终为O(nlogn)，但是需要额外的内存空间

若将两个有序表合并成一个有序表，称为2-路归并

自底向上的归并排序比较适合用链表组织的数据

自顶向下的归并排序优化

  • 对小规模子数组使用插入排序
    • 递归会是小规模问题中方法调用频繁
  • 测试数组是否已经有序
    • 当将两个子序列分别排序后，判断if arr[mid] <= arr[mid+1]那么就认为数组已经有序，并跳过merge方法

快速排序 Quick Sort

快速排序采用分治法（Divide and conquer）策略将一个串行（list）分成两个字串行（sub-lists）。本质上，快速排序是在冒泡排序基础上的递归分治法

快速排序的基本思想是通过一趟排序将记录通过pivot（主元、基准、轴点），将记录分隔成两部分，其中pivot前半部分的元素均比后半部分小，则继续分别对这两部分进行排序，以达到整个序列有序。

对于迭代版的快速排序，即通过一个栈来模拟系统调用栈，每次压入左右边界值即可。

算法步骤

  1. 从数列中挑出一个元素，称为pivot
  2. 分区（切分，partition）操作：将所有元素比pivot小的放在pivot前面，所有元素比pivot大的放在pivot后面（相同的数可以放在任一边）。在这个分区操作结束后，该pivot就处于数列的中间位置。
  3. 递归地将小于pivot的子序列和大于pivot的子数列进行排序

最快情况

快速排序最好的情况是每次都能正好将数组对半切分

最坏情况

快速排序最坏运行情况是O(n^2)，比如顺序数列的快排，但它的平摊期望时间是 O(nlogn)，且O(nlogn) 记号中隐含的常数因子很小，比复杂度稳定等于 O(nlogn) 的归并排序要小很多。所以，对绝大多数顺序性较弱的随机数列而言，快速排序总是优于归并排序。

其他话

快速排序的特点为，快排是原地排序（只需要很小的辅助栈），且长度为N的数组排序所愮的时间和NlgN成正比

保持随机性

  • 将数组元素的顺序打乱。或在partition中随机选择一个切分元素

处理切分元素值相同情况

  • 左侧扫描最好是遇到大于等于切分元素值时停下来，右侧扫描是遇到小于等于切分元素值停下来。这样可能会有不必要的一些等值元素交换，但是能够避免算法的运行时间变成平方级（包含大量重复值的数组情况）

算法改进

切换到插入排序

  • 对于小数组，快速排序比插入排序慢
  • 因为递归，快速排序的sort在小数组中也会调用自己

将if(hi <= lo) return;改进成if(hi <= lo + M){InsertionSort.sort(arr, lo, hi); return;}

三取样切分

使用子数组的一小部分元素的中位数来切分数组。取样大小为3，并用大小居中的元素切分

可将取样元素放在数组末尾作为哨兵，来去掉partition中的数组边界测试

堆排序 Heap Sort

堆排序是利用堆的数据结构进行的排序算法。堆积是一个近似完全二叉树的结构，即子结点的键值或索引总是小于（或者大于）其父结点。

  • 大顶堆：每个结点的值都大于或等于其子结点的值，在堆排序中算法用于升序排序
  • 小顶堆：每个结点的值都小于或等于其子结点的值，在堆排序中算法用于降序排序

堆排序的平均时间复杂度为O(nlogn)

堆的介绍

堆的存储是用数组来存放的，因此可以根据数组的索引来访问堆的元素。

对于堆的存放有两种方式

  1. 使用下标为0的数组空间（根节点放在arr[0]）
    • 对于某个元素的下标为k，其父节点的下标为(k-1)/2，其左孩子的下标为(2k+1)，右孩子的下标为(2k+2)
  2. 不使用下标为0的数组空间，从下标1开始存放堆元素（根节点放在arr[1]）
    • 对于某个元素的下标为k，其父节点的下标为k/2，其左孩子的下标为2k，右孩子的下标为2k+1

在访问某个结点的右孩子时，需要注意其是否存在右孩子

下面操作是基于下标从1开始存放堆元素的上浮与下称操作

由下至上的堆有序化（上浮swim）

某个结点比它的父节点更大而打破了堆的有序状态。

通过不断的将该结点与其父节点交换，直到其父节点大于该节点，或者该节点成为了堆顶

由上至下的堆有序化（下沉sink）

某个结点比它的两个子节点或是其中之一更小而打破堆的有序状态。

通过将该结点与其两个子结点中较大者交换来恢复堆，交换直到该结点的子结点都比它小或是达到堆的底部

算法步骤

这里进行从左到右，从小到大的升序排列。这里选择根节点从0开始存放

  1. 堆的构建（建堆）：根据升序降序需求选择大顶堆或小顶堆，这里选择大顶堆

    1. 从最后一个非叶子结点（最后一层的最后一个叶子结点的父节点）开始，进行下沉操作
    2. 从右到左，从下到上对每个非叶子结点进行下沉操作，最后一次为对根节点进行下沉操作

    （建堆完成后，堆顶为数组中最大元素）

  2. 下沉排序中销毁堆

    1. 取出堆顶元素，与堆的最后一个元素交换
    2. 堆的长度减一
    3. 直到堆中只剩下一个元素中，停止

其他话

堆排序无法利用缓存：数组元素很少和相邻的其他元素进行比较，因此缓存未命中的次数要远远高于大多数比较都在相邻元素间

计数排序 Counting Sort

计数排序的核心在于将待排序的数列的元素值，转化成额外开辟的数组空间的键。其时间复杂度为O(n)

计数排序的要求

  • 待排序的元素必须是整数
  • 待排序的元素的取值要在一定的范围内，并且比较集中

满足以上两点才能发挥计数排序的优势

算法步骤

  1. 找出原数组的最大值
  2. 创建一个新数组，其数组元素默认值均为0，长度为最大值+1
  3. 遍历原数组，将原数组元素值作为新数组的下标索引，该元素在原数组出现的次数作为新数组的值
  4. 根据新数组，反向填充原数组。在新数组中值不为0的元素的下标填入原数组，并且新数组值自减，直至为0
  5. 按顺序遍历原数组和新数组。

算法改进

原数组元素的最小元素可能很大，会浪费用来统计的新数组前部分的空间，因此我们只需要对于新数组声明的大小为max-min+1的长度即可（可以理解为将元素值向左移动min位，使得min映射到0）

桶排序 Bucket Sort

桶排序是计数排序的升级版。计数排序可以看作每个桶只储存相同的元素，而桶排序中每个桶储存一定范围的元素。通过映射函数，将待排序数组中的元素映射到不同的桶中，接着对每个桶中的元素进行排序，最后将非空桶中的元素放入原序列中。

划分多个范围相同的区间，每个子区间子排序，最后合并

  • 桶排序需要尽量保证元素分散均匀
  • 对于桶中元素的排序，选择何种比较算法对于性能提升影响至关重要

最快情况

输入的数据能够均匀的分配到每一个桶中

最坏情况

输入的数据都分配到同一个桶中

其他话

对于桶中，可以使用链表，再排序的时候用插入排序。也可以递归桶排序。或者存放数组，待取出时，先排序再取出

基数排序 Radix Sort

基数排序的原理是将整数按位分割成不同的数字，然后按每个位数分别比较。

基数排序也能用于名字、日期和特定格式的浮点数

算法步骤

  1. 取出数组中的最大值，获取其位数
  2. 从低位开始，根据低位的数值进行排序，再放入原数组
  3. 向高位进一位，继续排序
