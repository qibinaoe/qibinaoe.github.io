Java核心设计 第八章 泛型程序设计

泛型程序设计

泛型设计(generic programming) : 编写的代码可以对多种不同类型的对象重用

类型参数(type parameter) ArrayList<String> files = new ArrayList<>();

在Java增加泛型类之前，泛型程序设计是用继承实现的，存在两个问题

  • 当获取一个值时，必须进行强制类型转换
  • 没有错误检查，可以向数组列表中添加任何类的值

在使用泛型类和类型参数后

  • 编译器可以通过类型参数来知道返回值的类型，因此不需要进行强制类型转换
  • 在添加元素（ArrayList中的add()）时，编译器知道插入元素的类型，比Object类型安全。编译器可以检查，防止插入错误类型的对象

通配符类型(wildcard type)

泛型类

泛型类(generic class) :有一个或多个类型变量的类

  • 类型变量 T ，用尖括号(<>)括起来，放在类名后面

  • 泛型类可以有多个类型变量 public class Pair<T,U> {...}

  • 类型变量在整个类定义中，用于

    • 指定方法的返回类型
    • 字段的类型
    • 局部变量的类型

类型变量使用大写字母

  • 变量E表示集合的元素类型
  • K和V分别表示表的键和值的类型
  • T（和相邻的字母U和S）表示”任意类型“

泛型类相当于普通类的工厂

  • 用具体的类型替换类型变量来**实例化(instantiate)**泛型类型

泛型方法

泛型方法可以在普通类中定义也可以在泛型类中定义

**类型变量**的位置

  • 放在修饰符（这里指public static）的后面
  • 放在返回类型的前面（这里的返回类型是T）

调用一个泛型方法

  • 可以把具体类型包为在尖括号中，放在方法名前面
    • String middle = ArrayAlg.<String>getMiddle("Jogn","CAt","Dog");
  • 大多数情况下，编译器有足够信息推断出想法，将参数的类型和泛型类型T进行匹配进行推断，可以省略<String>类型参数
    • String middle = ArrayAlg.getMiddle("Jogn","CAt","Dog");

类型变量的限定

有时，类或方法需要对类型变量加以约束(例如保证方法中的某些方法能够正常调用)，可以通过对类型变量T设置一个**限定(bound)**来实现

使用关键字extends

记法 <T extends BoundingType>

  • T表示限定类型（BoundingType）的子类型（subtype）
  • T和限定类型可以是类也可以是接口
  • 一个类型变量(T)或通配符可以有多个限定(Bound)
    • T extends Comparable & Serializable
    • 限定类型（BoundingType）用”&”分隔
    • 类型变量（T)用逗号分隔
  • 在限定类型中可以拥有多个接口作为超类型，但最多有一个限定可以是类。
    • 如果一个类作为限定，它必须是限定列表中的第一个限定

Java核心技术——第五章 继承

类、超类和子类

  • 继承(inheritance)

    • “is-a”关系是继承的一个明显特征

    • Java用关键字extends表示继承

    • Java中所有继承都是公共继承，没有C++中的私有继承和保护继承

    • 新类派生于一个已存在的类

      • 已存在的类称为
        • 超类 superclass
        • 基类 base class
        • 父类 parent class
      • 新类称为
        • 子累 subclass
        • 派生类 derived class
        • 孩子类 child class

    • 通过扩展超类定义子类的时候，只需要支出子类与超类的不同之处。

  • 覆盖(override)方法

    • 超类中的有些方法对于子类不适用需要提供新的方法覆盖

      • 注意：子类不能直接访问超类的私有字段(private)，因此可以通过访问器来获取，但是如果我们重写的方法就是访问器的话，需要加上super，否则会一直循环调用自身以致程序崩溃

      • super和this概念不同

        • this是对象的引用(隐式参数)
        • super不是对象的引用
          • 不能将值super赋给另一个对象变量
          • super只是一个指示编译器调用超类方法的特殊关键字

  • 子类构造器

    • 由于子类的构造器不能访问父类的私有字段，所以必须通过一个构造器来初始化私有字段
    • 利用super语法来调用父类的构造器
      • 注意：使用super调用构造器的语句必须是子类构造器的第一条语句
    • 如果子类构造器没有显示地调用超类的构造器，将自动调用超类的无参数构造器
    • 如果超类没有无参数构造器，并且子类的构造器没有显示地调用超类的其他构造器，Java编译器会报错
    • this与super
      • this两个含义
        1. 指示隐式参数的引用
        2. 调用该类的其他构造器
      • super
        1. 调用超类的方法
        2. 调用超类的其他构造器
    • 调用构造器的语句只能作为另外一个构造器的第一条语句出现
    • 多态与动态绑定
      • 一个对象变量(例如下面代码中的e)可以指示多种实际类型的现象叫做多态(polymorphism)
      • 在运行时能够自动地选择适当的方法，称为动态绑定(dynamic binding)
      • 虽然下面的for each循环中的e声明为Employee,但是e既可以引用Employee类型的对象，也可以引用Manager类型的对象
      • 虚拟机知道e实际引用的对象类型，因此能够正确地调用相应的方法
      • Java中，动态绑定时默认的行为
      • 如果不希望让一个方法是虚拟的，可以将它标记为final

    • 继承层次(inheritance hierarchy)：由一个公共超类派生出来的所有类的集合

    • Java不支持多重继承

    • 多态

      • “is-a”规则可以用来判断是否应该将数据设计为继承关系，其指出子类的每个对象也是超类的对象(经理都是员工)，但是反之则不然

      • “is-a”规则另一种表述：替换原则(substitution principle)

      • Java对象变量是多态(polymorphic)的

  • 子类覆盖超类方法

    • 子类方法不能低于超类方法的可见性
      • 如果超类方式是public，那么子类的方法必须也要声明为public。如果子类方法遗漏掉了public修饰符，则编译器会报错

  • 方法调用过程

    • 调用x.f(args),其中x为C类的一个对象
      1. 编译器查看对象的声明类型和方法名。编译器会一一列举C类中所有方法名为f的方法和其超类中所有名为f且可访问的方法
      2. 编译器确定方法调用中提供的参数类型。在所有名为f的方法中，如果存在一个与所提供参数类型完全匹配的方法，就调用该方法。这个过程称为*重载解析(overloading resolution)*。如果编译器没有找到与参数类型匹配的方法，或者发现经过类型转换后有多个方法与之匹配，编译器会报错。
      3. 静态绑定(static binding):如果是private方法,static方法,final方法或构造器，则编译器可以准确知道应该调用哪一个方法；动态绑定:要调用的方法依赖于隐式参数的实际类型，必须要运行时使用动态绑定
      4. 假设程序运行并采用动态绑定调用方法，则虚拟机调用与x所引用对象的实际类型对应的那个方法。如果该类没有定义这个方法，则在父类中找。虚拟机预先为每个类计算了一个*方法表(method table)*，其中列出了所有方法的签名和要调用的实际方法。在真正调用方法时，虚拟机仅查找该表、

  • final类

    • 需求：如果想阻止人们用某个类来定义子类

    • 不允许扩展的类称为final类(定义类的时候使用final修饰符)

    • 例如阻止人们派生Executive类的子类

  • final方法

    • 类中的某个特定的方法也可以声明为final
    • 子类不能覆盖这个方法
    • final类中的所有方法自动成为final方法

  • 字段也能声明为final，final字段表示构造对象后就不能再改变值了。final类只有其中的方法自动成为final，不包括字段

  • 将方法或类声明为final的主要原因是确保它们在子类不会改变语义

    • Calendar类中的getTime()和setTime()方法声明为final。意味着Calendar类设计者负责实现Date类与日历状态的转换，不允许子类添乱
    • String类为final类，如果有一个String引用，则它引用一定是String对象

  • 内联(inlining)

  • 如果一个方法没有被覆盖(重写)而且很短，则编译器能够对它进行优化处理

  • 类的对象引用的强制类型转换

    • 用一对圆括号将目标类括起来，并放置在需要转换的对象引用之前

    • 使用对象引用的强制类型转换的唯一原因： 要在，暂时忽视对象的实际类型之后，使用对象的全部功能

      • 暂时忽视的情况：在staff中，某些元素是普通员工(Employee)，因此经理元素(Manager)需要暂时忽视它的实际类型
      • 要使用对象全部功能： 将staff数组中引用Manager对象的元素复原成Manager对象

    • 每个对象变量都有一个类型

      • 类型描述这个变量所引用的以及能够引用的对象类型

    • 一个值存入变量时，编译器会检查是否承诺过多

      • 子类的引用赋值给超类的变量，是允许的

    • 超类的引用赋值给子类变量时，是不可以的，承诺过多，必须进行强制类型转换

    • 如果试图在继承链上，进行向下的强制类型转换，并且谎报对象内容，会产生ClassCastException的异常

    • String c = (String) staff[1];会产生编译错误

    • 总之

      • 只能在继承层次内进行强制类型转换
      • 在超类强制转换成子类之前，应该使用instanceof进行检查

    • 因为实现多态性的动态绑定机制，能够自动找到正确的方法，很多情况不需要进行强制类型转换

    • 只有在使用子类(如Manager)中特有的方法才需要进行强制类型转换

  • 抽象类

    • 包含一个或多个抽象方法的类，类本身必须声明为抽象的

    • 抽象类还可以包含字段和具体方法

    • 通过引用一个公共超类，将方法放在继承层次结构中更高层

    • 而当高层的类中的方法需要根据实际情况实现时(例如这里的getDescription())，可以定义成抽象方法，就不需要实现这个方法，留到继承它的类去实现。

    • 扩展抽象类的两种情况

      • 子类保留抽象类的部分方法，或者是所有抽象方法都仍未定义，因此子类也应该标记为抽象类
      • 子类实现定义的全部方法，那么子类就不是抽象的了

    • 即使不含有抽象方法，也可以将类声明称抽象类

    • 抽象类不能实例化

    • 可以创建具体子类的对象(父类是抽象的，但是本身不是抽象的)、

    • 可以定义抽象类的对象变量

      • Person p = new Student("dog","math")
      • 这个定义为抽象类的对象变量只能引用非抽象子类的对象

    • 使用抽象类一个应用就是能够通过直接访问子类中实现抽象类的方法，方便处理

      • 如果这里将Person超类中的getDescription省略掉的话，就不能在变量p上调用getDescription方法了。

  • 受保护访问 protected

    • 希望限制超类中的某个方法只允许子类访问
    • 希望允许子类的方法能够访问超类的某个字段
    • Java中，保护字段只能由同一个包中的类访问
      • 避免滥用保护机制，不能通过派生子类来访问受保护字段

  • 访问控制修饰符

    • 仅对本类可见——private
    • 对外部完全可见——public
    • 对本包和所有子类可见——protected
    • 对本包可见——默认，不需要修饰符

  • Object:所有类的超类

    • Java中每个类都扩展了Object

    • 可以使用Object类型的变量引用任何类型的对象

    • Java中只有基本类型(primitive type)不是对象

      • 数值、字符、布尔类型都不是对象

  • 所有的数组类型，不管是对象数组还是基本类型的数组都扩展了Object类

  • 泛型数组列表

    • Java允许在运行时确定数组的大小

    但是声明这种数组不能在运行的时候动态改变数组的大小，一旦确定了数组的大小，就不容易改变它

    • ArrayList 数组列表

      • 类似数组，但是能够在添加或删除元素时，自动地调整数组容量

      • ArrayList是一个有类型参数(type parameter)的泛型类(generic class)

        • 用一对尖括号将类名括起来追加到ArrayList后面来指定数组列表保存的元素对象的类型

      • 声明和构造

        • ArrayList<Employee> staff = new ArrayList<Employee>();
        • 可以省去右边的类型参数：ArrayList<Employee> staff = new ArrayList<>();

      • 菱形语法 <>

        • 结合new操作符使用菱形语法
          • 编译器检查新值要做什么
            • 赋值给一个变量
            • 传递给某个方法
            • 从某个方法返回
          • 编译器检查这个变量、参数、方法的泛型类型，将这个类型放在<>上
        • 例如 new ArrayList<>()将赋值给一个类型为ArrayList<Employee>的变量，泛型类型为Employee

      • add方法 将元素添加到数组中

        • staff.add(new Employee(...));
        • 数组列表(ArrayList)管理着一个内部的对象引用数组
          • 调用add而内部数组已经满的时候，ArrayList会自动地创建一个更大的数组，并将所有的对象从较小的数组中拷贝到较大的数组中

      • ensureCapacity方法

        • 已经知道或能够估计出数组可能存储的元素数量，可以在填充之前调用ensureCapacity
          • staff.ensureCapacity(100)
          • 这样就不会带来开销很大的重新分配控件

      • 初是容量传递给ArrayList

    • ArrayList<Employee> staff = new ArrayList<>(100);

      • trimToSize

        • 将储存块的大小调整为当前保存当前元素数量所需要的储存空间。垃圾回收器将回收多余的储存空间

- 赋值操作`a = b`将会构造一个与b长度相同的新向量a，并将所有的元素由b拷贝到a

        • 改变第i个元素 set
          • staff.set(i,harry);

        • 得到元素
          • staff.get(i);

      • 在指定位置插入元素 void add(int index, E obj)

        • 后移元素，并将obj插入指定索引位置

      • 从指定位置删除元素 E remove(int index)

    • 删除索引位置的元素，并将后面的所有元素前移，返回所删除元素

      • 注意ArrayList在中间插入和删除元素的效率很低
      • for each 循环遍历数组列表内容
      • 虚拟机中没有类型参数
      • SuppressWarnings("")

  • 对象包装器和自动装箱

    • 所有的基本类型都有一个与之对应的类(包装器 wrapper)

      • Integer
      • Long
      • Float
      • Double
      • Short
      • Byte
      • Character
      • Boolean

    • 包装器类是不可变的

      • 里面储存的字段声明为final
      • private final int value;

    • 包装器类是final类，不能派生它们的子类

      • public final class Integer extends Number implements Comparable<Integer>

    • 尖括号中的类型参数不允许是基本类型

  • ArrayList<Integer>的效率远远低于int[]数组

    • 自动装箱（autoboxing)

      • list.add(3) 自动变成list.add(Integer.valueOf(3))

      • 将Integer对象赋给int值时，会自动地拆箱

      • int n = list.get(i) 自动转换成 int n = list.get(i).intValue()

    • 算术表达式自动装箱和拆箱

  • 包装器比较

    • ==运算符可以应用于包装器对象，但是是检测是否是相同的内存位置

      自动装箱规范要求boolean、byte、char <= 127，介于-128和127之间的short和int会被包装到固定的对象中，因此小于等于127的包装器的==比较是true

  • 表达式中混用Integer和Double类型，Integer值会拆箱，提升到double，再装箱为Double

    • 装箱和拆箱是编译器的工作

    • int intValue() 返回Integer对象的值作为int

  • 参数数量可变的方法

    • 提供参数数量可变的方法(变参(varargs)方法)

    • System.out.printf("%d",n);

    • System.out.printf("%d %s",n, "widgets");

      这里的 ...是Java代码的一部分，表明这个方法可以接受任意数量的对象,除format外

      Object ... args其实是一个Object[] 数组，保存所有其他参数，如果调用者提供的是基本类型的值，则会自动装箱

  • 枚举类

    • public enum Size {SAMLL, MEDIUM, LARGE, EXTRA_LARGE}
      • 声明定义的类型是一个类，它有四个实例，只有这四个实例，所以不可能再构造新的对象
    • 比较两个枚举类型的值，直接使用==就可以
    • 可以为枚举类型添加构造器，方法，字段

    • 所有枚举类型都是Enum类的子类，继承了很多方法

      • toString

        • System.out.println(Arrays.toString(Size.values())); //[SMALL, MEDIEUM, LARGE, EXTRA_LARGE]

      • ordinal()方法返回声明中枚举常量的位置，位置从0开始计数

      • int compareTo(E other)

        • 返回结果是 self.ordinal - other.ordinal

    • 反射库(reflection library)提供丰富且精巧的工具集，可以用来编写能够动态操纵Java代码的程序

`forName`方法会抛出一个*检查型异常(checked exception)*,因此需要提供一个*异常处理器(exception handler)*

      • 使用newInstance方法来构造一个实例

      • 检查型(checked)异常

    • 将公共操作和字段放在超类
    • 不要使用受保护的字段
    • 使用继承实现”is-a”关系
      • 如果不满足这种关系，就不要使用，例如父类有的某些字段和方法，在子类中不允许存在

    • 使用多态，而不要使用类型信息

Java核心技术—对象与类

发表于 2020-11-11 更新于 2020-11-30 阅读次数： Valine：

Java核心技术——第四章 对象与类

  • 面向对象设计(Object-oriented programming ,OOP)

  • 创建类的实例(instance): 由类class(构造对象的模板或蓝图)构造construct对象的过程

  • 封装(encapsulation)的关键在于程序只能通过对象的方法与对象数据进行交互，换句话说，绝对不能让类中的方法直接访问其他类的示例字段(instance field)

  • 对别对象和对象变量的概念

    • Date deadline; 这是一个对象遍历啊你给，不是一个对象，并且没有引用任何对象
    • 对象变量初始化的方法
      1. 引用一个新构造的对象 deadline = new Date();
      2. 引用一个已存在的对象 deadline = birthday;
    • 对象变量没有实际包含对象，而是引用一个对象
    • new操作符返回值是一个引用
    • 所有的Java对象都存储在堆中。一个对象包含另一个对象变量时，它只是包含着另一个堆对象的指针
    • Java中必须使用clone方法来获得对象的完整副本

  • UTC(Coordinated Universal Time) 国际协调时间

  • GMT(Greenwich Mean Time) 格林尼治时间

  • 访问器方法(accessor method) 只访问对象而不修改对象的方法

  • 修改器方法(mutator method) 访问对象后，对象的状态发生改变

  • 一个源文件(.java)只能由一个公共类，但可以任意数目的非公共类

    • 在编译的时候会分别为每个类生成.class文件

  • 构造器

    • 构造器结合new运算符调用
    • 不能对一个已经存在的对象调用构造器来达到重新设置实例字段的目的
    • 构造器没有返回值

  • 不要在构造器中定义与实例字段同名的局部变量

  • 隐式参数与显示参数

    • employee.raiseSalary(1000);
      • raiseSalary方法有两个参数
        1. 隐式参数： employee出现在方法前的Employee类型对象（方法调用的目标或接收者）
        2. 显示参数： 1000位于方法方法名后面括号中的数值
    • 在每个方法中，关键字this指示隐式参数
      • this.salary += raise;

  • Java中所有的方法都必须在类的内部定义(不同于C++),但这不表示它们是内联方法。是否将某个方法设置为内联方法是Java虚拟机的任务。即时编辑器会监视那些简短的、经常调用而且没有被覆盖的方法调用，并进行优化。

  • 访问器方法

    • 不要编写返回可变对象引用的访问器方法
    • 如果需要返回一个可变对象的引用，应该先对其克隆(clone)再返回。return (Date)hireDay.clone();

  • 基于类的访问权限

    • 方法可以访问调用这个方法的对象的私有数据
    • 一个方法可以访问所属类的所有对象的私有数据(可读可修改)

  • final实例字段

    • 实例字段定义为final
      • 这些字段必须在构造对象时初始化，即每一个构造器执行之后，这些字段的值已经设置，并且以后不能再修改这个字段
    • final修饰符对于类型为基本类型或不可变类(类中所有方法都不会改变其对象)的字段(String)
    • final修饰可变的类会造成混乱(StringBuilder)，因为final只表示储存在final修饰的变量的对象引用不会再知识另一个不同的对象，但是这个对象可以更改

  • 静态字段

    • 一个字段定义为static,则每个类只有一个这样的字段
      • 这个类的所有实例都共享这个static修饰的静态字段
      • 即使没有这个对象，静态字段仍然存在
      • 静态字段属于类，不属于任何单个对象
    • 对于非静态的实例字段，每个对象都有自己的副本

  • 静态常量

    • public static final double PI = 3.14159265358979323846;
    • public static final PrintStream out = ...;
    • 在System类中，有个setOut方法能够修改System.out为不同的流，这个方法能够修改final修饰变量的值的原因是因为这是个原生方法,不是在Java语言实现的，绕过Java语言的访问控制机制。

  • 静态方法

    • 不在对象上执行的方法 Math.pow(x, a), 由于不使用任何对象，因此它没有隐式参数
    • 静态方法没有this参数
    • 静态方法不能访问实例字段，但是可以访问静态字段
    • 非静态方法也也可以访问静态字段 这里的static可以省略，但这样一来就得通过对象引用来调用方法

  • 工厂方法

    • 静态工厂方法构造对象
      • 无法命名构造器
        • 希望有两个不同的名字得到不同的实例
      • 使用构造器，无法改变所构造对象的类型

  • 每一个类都可以有一个main方法

    • 对类进行的单元测试的技巧

  • 方法参数

    • 按值调用(call by value)

      • 接受调用者提供的值
      • 方法不能修改按值传递的变量的值

    • 按引用调用(call by reference)

      • 接受的是调用者提供的变量地址
      • 方法可以修改按引用传递的变量的值

    • Java总是采用按值调用

      • 方法得到的是所有参数值的一个副本，方法不能修改传递给它的任何参数变量的内容

      • 方法参数类型

        • 基本数据类型(数字、布尔值)
        • 对象引用(注：对象引用也是按值传递，不是按引用传递)

      • 但是通过对象引用能够修改对象的参数状态，因为方法得到的是对象引用的一个副本，原来对象引用和副本都引用同一个对象

      • 证明对象引用不是按引用传递

  • 对象构造

    • 重载(overloading)
      • 相同名字，不同参数。(不仅适用于构造器，Java允许重载任何方法)(注意区分重载(overload)和覆盖(override)区别)
    • 重载解析(overloading solution)
      • 查找匹配的过程
    • 方法的签名: 方法名+参数类型
      • 注意，返回值不是签名的一部分

  • 默认字段初始化

    • 构造器没有显示为字段设置初值，会被自动赋为默认值
      • 数值为0
      • 布尔值为false
      • 对象引用为null
    • 注意：方法中的局部变量必须明确地初始化，否则会报错

  • 无参数构造器

    • 写类的时候没有编写构造器，会提供一个无参数构造器
      • 所有实例字段设置为默认值
    • 仅当类没有任何其他构造器的时候，才会得到一个默认的无参数构造器
      • 如果类提供了至少一个构造器，但是没有提供无参数的构造器，那么构造对象时如果不提供参数就是不合法

  • 显式字段初始化

    1. 类定义中直接为任何字段赋值

      • 执行构造器之前先完成赋值操作

      • 希望将某个字段设置为同一值

    2. 利用方法调用初始化一个字段

  • this关键字调用另一个构造器

    • 构造器的第一个语句形如this(...)，构造器将调用同一个类的另一个构造器

  • 初始化块(initialization block)

    • 首先运行初始化块，再运行构造器的主体部分
    • 通常将初始化代码放在构造器里
    • 建议将初始化块放在字段定义之后

  • 调用构造器的具体处理步骤

    1. 如果构造器第一行调用了另一个构造器，则基于所提供的参数执行第二个构造器
    2. 否则
      1. 所有数据字段初始化为其默认值
      2. 按照类声明中出现的顺序，执行所有字段的初始化方法和初始化块
    3. 执行构造器主题代码

  • 静态字段初始化

    1. 提供初始值

      • private static int nextId = 1;

    2. 静态的初始化块、

  • 包

    • 一个类可以使用所属包的所有类，以及其他包的公共类
    • 在包中定位类是编译器(compiler)的工作。
    • 类文件中的字节码总是使用完整的包名引用其他类
    • 在Java中，通过显式地给出类名(java.util.Date)，可以完全避免使用import机制

  • 静态导入

    • import语句允许导入静态方法和静态字段，而不只是类
    • 源文件顶部添加import static java.lang.System.*;
      • 就可以使用System类的静态方法和静态字段，不必加类名前缀
      • out.println("Godd bye");
      • exit(0);

  • 包访问

    • public: 可以由任意类使用
    • private: 只能由定义它们的类使用。（提示，变量必须显示的标记为private，否则将默认为包访问，会破坏封装性）
    • 没有指定public和private的(类、方法、变量): 可以被同一个包的所有方法访问

  • 编译器查找文件

    • 如果引用了一个类而没有指定这个类的包(不是这样Date deadline = new java.util.Date();),它首先查找包含这个类的包(通过import指令，来确定其中是否包含这个类)
    • 如果找到了一个以上的类，就会产生编译时错误(因为完全限定类名必须是唯一的，与import次序无关)

  • java.lang包总会默认导入

  • JAR(Java Archive)

    • JAR文件使用ZIP格式组织文件和子目录，可以使用任何ZIP工具查看JAR文件
    • Java归档文件
    • 可以包含类也可以包含图片和声音等其他类型的文件
    • 创建JAR文件
      • jar cvf jarFileName file1 file2...
        • 例如jar cvf CalculatorClasses.jar *.class icon.gif
    • jar程序的选项类似Unix tar命令的选项
    • 清单文件(mainfest)
      • 包含在JAR文件当中，一个特殊的META-INF的子目录的
      • 清单文件被命名为MAINFEST.MF
      • 清单文件被分成多个节
        • 第一节称为主节(main section)
        • 随后的条目用来指定命名实体的属性

  • 文档注释 javadoc

    • javadoc抽取信息，为一下特性编写注释

      • 模块
      • 包
      • 公共类与接口
      • 公共的和受保护的字段
      • 公共的和受保护的构造器及方法

    • 注释放置在所描述特性的前面

    • 注释以/**开始，以*/结束

    • 自由格式文本(free-form text)

    • 标记以@开始，如@since、@param

    • 自由格式文本第一句应该是一个概要性的句子，javadoc工具将这些句子抽取出生成概要页

    • 自由格式文本中可以使用html修饰符

      • <em> ... </em> 强调
      • <strong> ... </strong> 着重强调
      • <img ...> 图片
      • {@code ...}等宽代码

    • 类注释

      • 类注释必须放在import语句之后，类定义之前

    • 方法注释

      • 方法必须放在所描述的方法之前
      • 除了通用标记，还能使用以下标记
        • @param variable description
          • 给当前方法的参数部分添加条目
          • 可以占据多行
          • 一个方法的所有@param必须放在一起
        • @return description
          • 给当前方法添加返回部分
          • 可以跨多行
        • @throws class description
          • 表示方法可能抛出异常

    • 字段注释

      • 只需要对公共字段(通常指静态常量public static final)建立文档

    • 通用注释

      • @since text
        • 建立一个”since”(始于)条目
        • “text”可以是引入这个特性的版本任何描述
          • @since 1.7.1
      • @author name
        • 作者条目
        • 可以使用多个@author标记
      • version text
        • 版本条目
        • 对当前版本的任何描述
      • @see和@link
        • 可以使用超链接，链接到javadoc文档的相关部分或外部文档
        • @see reference
          • 将在”see also”部分增加一个超链接
          • 可用于类和方法
          • reference可以有以下选择
            1. package.class#feature label
              • 例如@see com.horstmann.corejava.Employee#raiseSalary(double)
              • 包名或包名和类名都可以省略，这样会定位于当前包或当前类
            2. <a href="...">label</a>
              • @see <a href="www.horstmann.com/corejava.html">The Core Java home page</a>
            3. "text"
              • @see "Core Java 2 volume 2"
        • {@link package.class#feature lable}

    • 包注释

      • 需要在每个包目录中添加一个单独文件
        1. 提供package-info.java的Java文件
          • 该文件包含一个初是的以/**和*/界定的javadoc注释
          • 后面是一个package语句
        2. 提供package.html的HTML文件
          • 会抽取<body></body>之间所有文本

  • 类设计技巧

    • 一定要保证数据私有
      • 绝对不要破坏封装性
    • 一定要对数据进行初始化
      • 不要依赖系统的默认值
      • 自己提供默认值
      • 所有构造器中设置默认值
    • 不要再类中使用过多的基本类型
      • 用其他类替换使用多个相关的基本类型
    • 不是所有的字段都需要单独的字段访问器和字段更改器
    • 分解有过多指责的类
    • 类名和方法名要能够体现出它的职责
    • 优先使用不可变类
      • 并发更改：多个线程试图同时更新一个对象
      • 如果类是不变的，可以安全地在多个线程间共享其对象
